# Power BI: Introduction to DAX (Data Analysis Expressions)

## Overview
DAX (Data Analysis Expressions) is a library of functions and operators that can be combined to build formulas and expressions in Microsoft Power BI, Analysis Services, and Power Pivot in Excel. As explored in the **Google Data Analytics Professional Certificate** and industry-standard Power BI workflows, DAX is what allows a Data Analyst to create new information from data already in your model.

---

## 1. Core Concepts: Calculated Columns vs. Measures
Understanding the difference between these two is the first step to mastering DAX.

### Calculated Columns
* **Context:** Row Context (calculated row by row).
* **Storage:** Stored in the model's memory (increases file size).
* **When to use:** When you need to use the result in a **Slicer** or as a **Row/Column header** in a visual.
* **Example:** Concatenating First Name and Last Name.

### Measures
* **Context:** Filter Context (calculated on the fly based on the filters in your visual).
* **Storage:** Not stored in the model; calculated at query time.
* **When to use:** For almost all numerical aggregations (Sum, Average, Count).
* **Example:** Calculating Total Sales or Year-over-Year growth.

---

## 2. DAX Syntax
A typical DAX formula looks like this:
`Total Sales = SUM(Sales[Revenue])`

* **Total Sales:** The name of the Measure.
* **SUM:** The DAX function.
* **Sales:** The Table name.
* **[Revenue]:** The Column name.

---

## 3. Essential DAX Functions

### Aggregation Functions
These functions summarize values from a column.
```dax
Total Revenue = SUM(Sales[Amount])
Average Order Value = AVERAGE(Sales[Amount])
Total Orders = COUNT(Sales[OrderID])
Unique Customers = DISTINCTCOUNT(Sales[CustomerID])
```

### Logical Functions
Used to evaluate conditions.
```dax
Sales Category = IF(Sales[Amount] > 1000, "High Value", "Standard")
Status Code = SWITCH(Sales[StatusID], 1, "Open", 2, "Pending", 3, "Closed", "Unknown")
```

### The CALCULATE Function (Most Important)
`CALCULATE` is the most powerful function in DAX. It evaluates an expression in a context that is modified by specific filters.
```dax
Total Sales in India = CALCULATE([Total Sales], Sales[Country] = "India")
Red Product Sales = CALCULATE(SUM(Sales[Amount]), Products[Color] = "Red")
```

---

## 4. Time Intelligence Functions
These allow you to manipulate data using time periods, which is essential for business reporting. These functions require a dedicated **Date Table** in your model.

### Year-to-Date (YTD)
```dax
Sales YTD = TOTALYTD([Total Sales], 'Date'[Date])
```

### Prior Year Comparison
Comparing current performance against the same period last year.
```dax
Sales Last Year = CALCULATE([Total Sales], SAMEPERIODLASTYEAR('Date'[Date]))
```

### Growth Calculations
```dax
YoY Sales Growth = 
VAR CurrentSales = [Total Sales]
VAR LastYearSales = [Sales Last Year]
RETURN
DIVIDE(CurrentSales - LastYearSales, LastYearSales, 0)
```

---

## 5. Best Practices for Writing DAX
* **Use Measures for Aggregations:** Always create a measure for sums and averages instead of dragging the raw column into a visual.
* **Table References:** Always include the Table Name when referencing a **Column** (`Sales[Revenue]`), but omit the Table Name when referencing a **Measure** (`[Total Sales]`). This makes your code much easier to read.
* **Format your DAX:** Use the `DIVIDE` function instead of the `/` operator to handle "Division by Zero" errors gracefully.
* **Variables (VAR/RETURN):** Use variables to break down complex formulas into readable steps and improve performance.
