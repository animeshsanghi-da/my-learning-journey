# Power BI: Creating Visuals (Cards, Slicers, Matrices, and Charts)

## Overview
Visualizations are the heart of Power BI. They transform raw data into a visual story that allows stakeholders to identify trends, outliers, and patterns at a glance. Effective visualization involves choosing the right chart type for the specific data story you are trying to tell.

---

## 1. Core Visual Types

### Cards and Multi-row Cards
Cards are used to display a single, high-level key metric (KPI). 
* **Use Case:** Total Sales, Total Profit, or Average Rating.
* **Best Practice:** Keep them large and clear at the top of your dashboard.

### Slicers
Slicers are on-canvas visual filters that allow users to narrow down the data shown in all other visuals on the report page.
* **Use Case:** Filtering by Year, Region, or Product Category.
* **Pro Tip:** Use the "Sync Slicers" pane if you want a selection on one page to apply to other pages in the report.

### Matrices and Tables
* **Tables:** Best for showing detailed, raw data in a grid format.
* **Matrices:** Similar to Excel Pivot Tables; they allow for multi-dimensional analysis with rows and columns, supporting "Drill Down" features.

---

## 2. Standard Charts and When to Use Them

| Visual Type | Best For... |
| :--- | :--- |
| **Bar / Column Charts** | Comparing categories (e.g., Sales by Region). |
| **Line Charts** | Showing trends over time (e.g., Monthly Revenue). |
| **Pie / Donut Charts** | Showing parts-of-a-whole relationships (use sparingly with < 5 categories). |
| **Scatter Charts** | Showing the relationship/correlation between two variables. |
| **Treemaps** | Visualizing hierarchical data or large sets of categories. |
| **Waterfall Charts** | Showing the cumulative effect of positive and negative changes (e.g., Budget vs. Actual). |

---

## 3. Enhancing Visual Interactivity

### Drill Down and Drill Through
* **Drill Down:** Allows users to move from a high-level category to a detailed one within the same visual (e.g., Year -> Quarter -> Month).
* **Drill Through:** Allows users to right-click a data point in one visual and navigate to a separate, detailed page filtered for that specific data point.

### Tooltips
Tooltips are the information boxes that appear when you hover over a data point.
* **Standard Tooltips:** Display the data values already in the visual.
* **Report Page Tooltips:** You can design a whole hidden page and set it as a tooltip, allowing you to show mini-charts when hovering over a main chart.

---

## 4. Formatting and Design Principles
A professional report must be clean and accessible.
* **Color Theory:** Use consistent colors. Use contrasting colors to highlight specific data points.
* **Data Ink Ratio:** Remove unnecessary gridlines, borders, or backgrounds that do not add information.
* **Conditional Formatting:** Just like in Excel, you can color-code your Power BI visuals based on performance (e.g., Green for target met, Red for below target).

---

## 5. Visual Query Logic (DAX Integration)
While Power BI is "drag-and-drop," you often use DAX to define what goes into a visual. 

**Example: Creating a dynamic title for a chart using DAX**
```dax
Dynamic Title = 
"Sales Performance for " & SELECTEDVALUE(Region[RegionName], "All Regions")
```
*Place this measure in the "Conditional Formatting" section of the Visual Title to make your dashboard react to slicer selections.*

---

## 6. The "Share" Phase (Google Data Analytics Alignment)
Following the **Google Data Analytics Lifecycle**, creating visuals is part of the **Share** phase. 
* **Accessibility:** Ensure your charts have high contrast for color-blind users and include "Alt-text" for screen readers.
* **Context:** Always provide a clear title and axis labels so the viewer doesn't have to guess what the data represents.
