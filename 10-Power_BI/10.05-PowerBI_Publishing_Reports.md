# Power BI: Publishing Reports and Dashboard Layouts

## Overview
[cite_start]The final stage of the Business Intelligence workflow is **Sharing and Acting**, as defined in the Google Data Analytics Professional Certificate[cite: 1, 3]. Publishing turns a static report into a collaborative business tool. This process involves moving your work from Power BI Desktop to the Power BI Service (Cloud), allowing for automated refreshes and mobile accessibility.

---

## 1. Preparing for Publication
Before clicking "Publish," you must ensure the report is optimized for end-users:
* **Report vs. Dashboard:** A *Report* is a multi-page document with detailed visuals. A *Dashboard* is a single-page "high-level" view created in the Power BI Service by pinning visuals from reports.
* **Canvas Settings:** Standard 16:9 for desktop or "Tooltip" size for hover-over details.
* **Selection Pane:** Ensure all visuals are named correctly for screen readers (Accessibility).
* **Tab Order:** Set the order in which users navigate through visuals using the "Tab" key.

---

## 2. The Publishing Workflow
1. **Sign In:** You must be signed into your Power BI Pro or Premium account in Power BI Desktop.
2. **Click Publish:** Located on the **Home** tab.
3. **Select Workspace:** Choose between "My Workspace" (personal) or a "Shared Workspace" (collaborative/departmental).
4. **Success Message:** Once uploaded, you receive a link to view the report in a web browser.

---

## 3. Creating Effective Dashboard Layouts
Dashboards should follow the **"F-Pattern"** or **"Z-Pattern"** of reading:
* **Top-Left:** Most critical KPIs (e.g., Total Revenue, Net Profit).
* **Middle:** Trending data (Line charts showing performance over time).
* **Bottom/Right:** Granular details or categorical breakdowns (Bar charts, Tables).

### Design Best Practices
* **Slicers:** Place filters (Date, Region, Category) at the top or left side for easy access.
* **White Space:** Do not overcrowd; use white space to separate different logical sections.
* **Color Consistency:** Stick to a professional theme or company branding. Use "Color Blind Safe" palettes.

---

## 4. Power BI Service Features (The Cloud Experience)
Once published, several cloud-only features become available:
* **Scheduled Refresh:** Configure a **Data Gateway** to automatically update your visuals when the underlying Excel/SQL data changes.
* **Mobile Layout:** Design a specific vertical view for users accessing the report via the Power BI phone app.
* **Q&A (Natural Language Query):** Users can type questions like "What was the total sales last month?" and Power BI will generate a visual response.
* **Alerts:** Set notifications on cards (e.g., "Email me if Profit drops below 10%").

---

## 5. Security and Distribution
* **Row-Level Security (RLS):** Restrict data access for certain users. [cite_start]For example, a Regional Manager in Indore should only see data for Madhya Pradesh[cite: 1, 3].
* **Workspaces:** Use for internal collaboration.
* **Power BI Apps:** The best way to distribute a finished set of dashboards and reports to a large group of viewers.
* **Publish to Web:** Generates a public link (Use with caution; this makes data visible to anyone on the internet).
