# Machine Learning: Training Models

## 1. Introduction to Supervised Learning
Machine Learning (ML) is the process of training an algorithm to identify patterns in data so it can make predictions on new, unseen data. In **Supervised Learning**, the model learns from a dataset that contains both input features and the correct output (labels).

### Core Concepts
* **Features (X):** The independent variables used to make a prediction (e.g., square footage, number of rooms).
* **Target (y):** The dependent variable we want to predict (e.g., house price).
* **Model Training:** The process where the algorithm adjusts its internal parameters to minimize the error between its predictions and the actual labels.

---

## 2. Linear Regression
Linear Regression is a foundational algorithm used to predict a continuous numerical value. It assumes a linear relationship between the input features and the target.

### Mathematical Intuition
The model fits a line defined by the equation:  
**y = mx + c** (where **m** is the slope/coefficient and **c** is the intercept).

### Implementation with Scikit-Learn
```python
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error, r2_score

# 1. Initialize the model
lin_reg = LinearRegression()

# 2. Train the model using the training set
lin_reg.fit(X_train_prepared, y_train)

# 3. Make predictions
predictions = lin_reg.predict(X_test_prepared)

# 4. Evaluate performance
mse = mean_squared_error(y_test, predictions)
r2 = r2_score(y_test, predictions)

print(f"Mean Squared Error: {mse}")
print(f"R-squared Score: {r2}")
```

---

## 3. Random Forest Regressor
Random Forest is an ensemble learning method that operates by constructing a multitude of **Decision Trees** during training. It is more robust than a single decision tree because it averages the results of many trees to improve predictive accuracy and control over-fitting.

### Key Advantages
* **Non-linearity:** Can capture complex relationships that Linear Regression might miss.
* **Feature Importance:** Automatically identifies which features are most influential in making predictions.
* **Robustness:** Handles outliers and missing values better than many simple models.

### Implementation
```python
from sklearn.ensemble import RandomForestRegressor

# 1. Initialize the model with hyperparameters
# n_estimators is the number of trees in the forest
forest_reg = RandomForestRegressor(n_estimators=100, random_state=42)

# 2. Train the model
forest_reg.fit(X_train_prepared, y_train)

# 3. Predict and Evaluate
forest_predictions = forest_reg.predict(X_test_prepared)
forest_mse = mean_squared_error(y_test, forest_predictions)
print(f"Random Forest MSE: {forest_mse}")
```

---

## 4. Model Evaluation Metrics
To understand how well a model is performing, we use specific statistical metrics:

| Metric | Definition | Interpretation |
| :--- | :--- | :--- |
| **Mean Squared Error (MSE)** | Average of the squares of the errors. | Lower is better; penalizes large errors heavily. |
| **Root Mean Squared Error (RMSE)** | Square root of MSE. | Represents error in the same units as the target variable. |
| **R-squared (R²)** | Proportion of variance explained by the model. | Ranges from 0 to 1; 1 is a perfect fit. |

---

## 5. Overfitting vs. Underfitting
One of the most critical aspects of training is finding the right balance of model complexity.



[Image of Overfitting vs Underfitting in Machine Learning]


* **Underfitting:** The model is too simple to learn the underlying patterns (High Bias). Performance is poor on both training and test data.
* **Overfitting:** The model learns the training data "too well," including the noise (High Variance). It performs great on training data but fails to generalize to new test data.

---

## 6. Cross-Validation
Instead of relying on a single train/test split, we use **K-Fold Cross-Validation** to get a more reliable estimate of model performance.


## 7. Hyperparameter Tuning
Hyperparameters are settings that are not learned by the model but are set by the analyst (e.g., the number of trees in a Random Forest). We use tools like `GridSearchCV` to find the best combination.

```python
from sklearn.model_selection import GridSearchCV

param_grid = [
    {'n_estimators': [3, 10, 30], 'max_features': [2, 4, 6, 8]},
]

grid_search = GridSearchCV(forest_reg, param_grid, cv=5,
                           scoring='neg_mean_squared_error',
                           return_train_score=True)

grid_search.fit(X_train_prepared, y_train)
print(f"Best Parameters: {grid_search.best_params_}")
```

---

**Summary:** Training a model is an iterative process of selecting an algorithm, training on processed data, evaluating against metrics, and tuning parameters to ensure the model generalizes well to real-world scenarios.
