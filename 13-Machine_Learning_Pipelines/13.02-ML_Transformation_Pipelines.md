# Machine Learning: Building Data Transformation Pipelines

## Overview
In a professional machine learning workflow, data rarely arrives in a format ready for training. Data Transformation Pipelines are essential for automating the journey from raw data to model-ready features. As covered in the **Google Data Analytics Professional Certificate** and the **Python Data Science Handbook**, a robust pipeline ensures consistency, prevents data leakage, and makes your experiments reproducible.

---

## 1. Why Use Pipelines?
* **Consistency:** Apply the exact same transformations to both training and testing data without manual repetition.
* **Scalability:** Easily handle new data by passing it through the established pipeline.
* **Prevention of Data Leakage:** Ensures that statistics used for scaling (like the mean or standard deviation) are only calculated from the training set and then applied to the test set.
* **Clean Code:** Encapsulates complex preprocessing logic into a single object.

---

## 2. Core Components of a Transformation Pipeline
A typical pipeline involves several stages, often handled by libraries like `Scikit-Learn`:

### A. Data Cleaning & Imputation
Handling missing values is the first priority.
* **Simple Imputer:** Replaces missing values with the mean, median, most frequent value, or a constant.
* **KNN Imputer:** Uses the k-Nearest Neighbors approach to predict and fill missing values based on similarity.

### B. Feature Scaling
Most Machine Learning algorithms (like Linear Regression or SVM) perform better when numerical features are on a similar scale.
* **StandardScaler:** Transforms data to have a mean of 0 and a standard deviation of 1.
* **MinMaxScaler:** Scales data to a fixed range, usually 0 to 1.

### C. Categorical Encoding
Converting text categories into numerical values that a model can interpret.
* **One-Hot Encoding:** Creates binary columns for each category (ideal for non-ordinal data like "City" or "Color").
* **Ordinal Encoding:** Assigns an integer to each category based on order (ideal for data like "Low", "Medium", "High").

---

## 3. Implementation with Scikit-Learn
Below is a conceptual implementation of a transformation pipeline using `ColumnTransformer` and `Pipeline`.

```python
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer

# 1. Define specific pipelines for different data types
num_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy="median")),
    ('std_scaler', StandardScaler()),
])

cat_pipeline = Pipeline([
    ('imputer', SimpleImputer(strategy="most_frequent")),
    ('one_hot', OneHotEncoder()),
])

# 2. Combine them into a single ColumnTransformer
# Assuming 'num_attribs' and 'cat_attribs' are lists of column names
full_pipeline = ColumnTransformer([
    ("num", num_pipeline, num_attribs),
    ("cat", cat_pipeline, cat_attribs),
])

# 3. Apply the transformation to your dataset
# housing_prepared = full_pipeline.fit_transform(housing)
```

---

## 4. Advanced Pipeline Techniques
* **Custom Transformers:** You can write your own classes to perform domain-specific transformations (e.g., combining two columns to create a new ratio feature).
* **Feature Selection:** Adding a stage in the pipeline to automatically select the most relevant features using techniques like SelectKBest.
* **Dimensionality Reduction:** Integrating PCA (Principal Component Analysis) to reduce the number of features while retaining variance.

---

## 5. Integrating with Model Training
The final step is to append the estimator (the model) to the end of the transformation pipeline. This allows you to call `.fit()` and `.predict()` on the raw input data.

```python
from sklearn.ensemble import RandomForestRegressor

# Create a final pipeline including the model
complete_ml_pipe = Pipeline([
    ("preparation", full_pipeline),
    ("model", RandomForestRegressor())
])
```
