# Object-Oriented Programming: The Four Pillars

## Overview
Object-Oriented Programming (OOP) is a programming paradigm built around the concept of "objects," which can contain data (attributes) and code (methods). Python is a multi-paradigm language, but its OOP capabilities are incredibly powerful for structuring large, complex data science applications and machine learning pipelines. 

The core of OOP rests on four fundamental pillars: **Encapsulation**, **Abstraction**, **Inheritance**, and **Polymorphism**.

---

## 1. Encapsulation
**Encapsulation** is the bundling of data (variables) and methods (functions) that operate on that data into a single unit or class. More importantly, it involves restricting direct access to some of the object's components. This is a crucial security and structural mechanism to prevent accidental data modification.

In Python, encapsulation is achieved using access modifiers (though they are more like strong conventions than strict enforcements like in Java/C++):
* **Public:** Accessible from anywhere. (e.g., `name`)
* **Protected:** Intended to be accessed only within the class and its subclasses. Denoted by a single underscore. (e.g., `_salary`)
* **Private:** Intended to be accessed *only* within the class. Denoted by a double underscore. Python uses "name mangling" to hide these. (e.g., `__bank_account`)

### Code Example: Encapsulation
```python
class Employee:
    def __init__(self, name, salary, account_number):
        self.name = name                 # Public attribute
        self._salary = salary            # Protected attribute
        self.__account_number = account_number # Private attribute

    # Getter method to securely access private data
    def get_account_number(self):
        return self.__account_number

    # Setter method to securely modify private data
    def set_account_number(self, new_account):
        if len(str(new_account)) == 10:
            self.__account_number = new_account
        else:
            print("Invalid account number length!")

emp = Employee("Animesh", 85000, 1234567890)
print(emp.name)                   # Works fine
print(emp._salary)                # Works, but bad practice
# print(emp.__account_number)     # Throws AttributeError!

# Secure Access
print(emp.get_account_number())   # Outputs: 1234567890
```

## 2. Abstraction
**Abstraction** means hiding the complex implementation details of a system and exposing only the essential features to the user. You don't need to know how the engine of a car works internally to drive it; you just need to know how to use the steering wheel and pedals.

In Python, we implement abstraction using abstract classes and methods via the built-in `abc` (Abstract Base Classes) module. An abstract class cannot be instantiated directly and usually contains at least one abstract method that *must* be implemented by its subclasses.

### Code Example: Abstraction
```python
from abc import ABC, abstractmethod

# Abstract Base Class
class Shape(ABC):
    @abstractmethod
    def area(self):
        pass # The implementation is hidden and deferred to subclasses

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
        
    # We MUST implement the area method, otherwise Circle cannot be instantiated
    def area(self):
        return 3.14 * (self.radius ** 2)

class Square(Shape):
    def __init__(self, side):
        self.side = side
        
    def area(self):
        return self.side ** 2

# obj = Shape() # This would throw an error! You cannot instantiate an abstract class.
c = Circle(5)
print(f"Circle Area: {c.area()}")
# Output: Circle Area: 78.5
```

## 3. Inheritance
**Inheritance** allows a new class (Child/Derived class) to inherit attributes and methods from an existing class (Parent/Base class). This promotes immense **code reusability** and establishes a natural, hierarchical relationship between classes.

**Types of Inheritance in Python:**
1. **Single Inheritance:** One child class inherits from one parent class.
2. **Multiple Inheritance:** One child class inherits from multiple parent classes.
3. **Multilevel Inheritance:** A child class inherits from a parent class, which in turn inherited from another parent class (Grandparent -> Parent -> Child).

The `super()` function is heavily used in inheritance to call methods or constructors from the parent class so you don't have to rewrite them.

### Code Example: Single & Multiple Inheritance
```python
# Parent Class 1
class DataAnalyst:
    def __init__(self, tools):
        self.tools = tools
        
    def analyze_data(self):
        print(f"Analyzing data using: {self.tools}")

# Parent Class 2
class Manager:
    def __init__(self, team_size):
        self.team_size = team_size
        
    def conduct_meeting(self):
        print(f"Conducting meeting with {self.team_size} members.")

# Child Class inheriting from BOTH (Multiple Inheritance)
class AnalyticsManager(DataAnalyst, Manager):
    def __init__(self, tools, team_size, department):
        # Initializing parent classes
        DataAnalyst.__init__(self, tools)
        Manager.__init__(self, team_size)
        self.department = department
        
    def show_role(self):
        print(f"Managing the {self.department} department.")

am = AnalyticsManager(["Python", "SQL", "Tableau"], 15, "Business Intelligence")
am.analyze_data()      # Inherited from DataAnalyst
am.conduct_meeting()   # Inherited from Manager
am.show_role()         # Defined in AnalyticsManager
```

## 4. Polymorphism
**Polymorphism** translates to "many forms." In OOP, it means that the same function name (or operator) can be used for different types, and it will behave differently depending on the object it is acting upon.

The most common ways to implement polymorphism in Python are:
1. **Method Overriding:** A child class provides a specific implementation of a method that is already provided by its parent class.
2. **Duck Typing:** "If it walks like a duck and quacks like a duck, it must be a duck." Python doesn't care about the strict type of an object; it only cares if the object has the specific method being called.

### Code Example: Polymorphism via Method Overriding
```python
class DatabaseConnection:
    def connect(self):
        print("Connecting to a generic database...")

class MySQLConnection(DatabaseConnection):
    # Overriding the parent's connect method
    def connect(self):
        print("Connecting to MySQL Database using sqlalchemy...")

class MongoDBConnection(DatabaseConnection):
    # Overriding the parent's connect method
    def connect(self):
        print("Connecting to MongoDB using pymongo...")

# Polymorphic behavior in action
def initialize_db(db_obj):
    db_obj.connect() # The exact behavior depends on the object passed

sql_db = MySQLConnection()
nosql_db = MongoDBConnection()

initialize_db(sql_db)   # Outputs: Connecting to MySQL Database using sqlalchemy...
initialize_db(nosql_db) # Outputs: Connecting to MongoDB using pymongo...
```
