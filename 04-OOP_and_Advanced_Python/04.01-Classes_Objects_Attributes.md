# Object-Oriented Programming: Classes, Objects, and Attributes

## Overview
Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around **data** (objects) rather than functions and logic. As your Python projects scale, OOP helps in managing complexity, reusing code, and modeling real-world entities accurately.

---

## 1. Core Concepts: Classes and Objects

### What is a Class?
A **Class** is a blueprint or a template for creating objects. It defines a set of attributes (variables) and methods (functions) that characterize any object instantiated from this class. 
* *Analogy:* A Class is like the architectural blueprint of a house.

### What is an Object?
An **Object** is an instance of a Class. Once a class is defined, you can create multiple objects from it, each with its own specific data.
* *Analogy:* If the class is the blueprint, the Object is the actual physical house built from that blueprint.

### Basic Syntax
To define a class in Python, use the `class` keyword followed by the class name (usually written in `PascalCase`).

```python
class Employee:
    # Class body goes here
    pass

# Creating objects (Instantiating)
emp1 = Employee()
emp2 = Employee()

print(type(emp1))  # Output: <class '__main__.Employee'>
```

---

## 2. Attributes: Class vs. Instance

Attributes are the variables that belong to a class or an object. Understanding the difference between Class Attributes and Instance Attributes is crucial.

### Class Attributes
These are variables defined directly inside the class body. They are shared across **all** instances (objects) of that class.

```python
class Employee:
    company = "Google"  # This is a class attribute

# Both objects share the same class attribute
emp1 = Employee()
emp2 = Employee()

print(emp1.company)  # Output: Google
print(emp2.company)  # Output: Google

# If we change the class attribute, it changes for all instances
Employee.company = "Alphabet"
print(emp1.company)  # Output: Alphabet
```

### Instance Attributes
These are variables that belong specifically to a single object. They are unique to each instance.

```python
class Employee:
    company = "Google"

emp1 = Employee()
emp1.name = "Animesh"     # Instance attribute specific to emp1
emp1.role = "Data Analyst"

emp2 = Employee()
emp2.name = "Rohan"       # Instance attribute specific to emp2

print(emp1.name)  # Output: Animesh
print(emp2.name)  # Output: Rohan
```

**Note on Precedence:** If an object has an instance attribute with the same name as a class attribute, the instance attribute takes priority when accessed through the object.

## 3. The `self` Parameter
When calling a method (a function inside a class) on an object, Python automatically passes the object itself as the first argument to the method. By convention, this parameter is named `self`.

```python
class Employee:
    company = "Google"
    
    # defining "Instance Method" (default) no need of any decorator
    def get_info(self):
        # 'self' refers to the object calling this method
        print(f"Employee {self.name} works at {self.company}")

emp = Employee()
emp.name = "Animesh"

# Python automatically translates this to Employee.get_info(emp)
emp.get_info()  
# Output: Employee Animesh works at Google
```

## 4. The `__init__` Method (Constructor)
Assigning instance attributes manually after creating an object is repetitive and prone to errors. Instead, we use the `__init__` method. 

The `__init__` method is a special "dunder" (double underscore) method, also known as a **constructor**. It is called automatically the exact moment an object is created.

```python
class Employee:
    company = "Google" # Class attribute
    
    # The Constructor
    def __init__(self, name, role, salary):
        # Setting instance attributes upon creation
        self.name = name
        self.role = role
        self.salary = salary
        print(f"Employee object created for {self.name}")

    def get_details(self):
        print(f"{self.name} is a {self.role} earning ${self.salary}.")

# Instantiating objects triggers the __init__ method automatically
emp1 = Employee("Animesh", "Data Analyst", 85000)
emp2 = Employee("Priya", "Data Scientist", 120000)

emp1.get_details()
# Output: Animesh is a Data Analyst earning $85000.
```

### Why is `__init__` so important?
1. **Ensures Data Integrity:** It forces you to pass the required data (like `name` and `role`) the moment the object is created, preventing objects from existing in an incomplete state.
2. **Cleaner Code:** It consolidates attribute assignment into a single block of code within the class template.
