# Exception Handling in Python (try, except, finally)

## Overview
No matter how well-written a program is, unexpected situations will occur during execution—like a user entering text when a number is required, a missing file, or a dropped database connection. In Python, these runtime errors are called **Exceptions**. Exception handling is a critical skill for Data Analysts and Engineers, as it prevents data pipelines and applications from crashing abruptly, allowing for graceful failure, logging, and recovery.

---

## 1. Syntax Errors vs. Exceptions
Before diving into handling exceptions, it's important to understand the difference between the two main types of errors in Python:
* **Syntax Errors:** Occur when the grammatical rules of Python are violated (e.g., missing a colon, unclosed parentheses). Python cannot even run the code until these are fixed.
* **Exceptions:** Occur *during* the execution of the program (runtime) when syntactically correct code fails. Examples include:
    * `ZeroDivisionError`: Dividing a number by zero.
    * `FileNotFoundError`: Trying to read a file that doesn't exist.
    * `ValueError`: Passing an argument with the right type but inappropriate value (e.g., `int("abc")`).
    * `TypeError`: Applying an operation to an inappropriate data type (e.g., adding a string and an integer).

---

## 2. The `try` and `except` Blocks
The primary mechanism for handling exceptions in Python is the `try...except` block. 
* **`try` block:** Contains the code that *might* cause an exception.
* **`except` block:** Contains the code that executes *only if* an exception occurs in the `try` block.

```python
# Basic Example
try:
    # This will raise a ZeroDivisionError
    result = 10 / 0
except ZeroDivisionError:
    print("Error: You cannot divide a number by zero!")
```

---

## 3. Catching Multiple Exceptions
In real-world data pipelines, a single block of code might fail for various reasons. You can handle different exceptions in different ways using multiple `except` blocks.

```python
try:
    num = int(input("Enter a divisor: "))
    result = 100 / num
    print(f"Result: {result}")
except ValueError:
    print("Data Error: Please enter a valid integer, not text.")
except ZeroDivisionError:
    print("Math Error: Cannot divide by zero.")
except Exception as e:
    # A generic catch-all for any other unexpected exception
    # 'e' captures the exception object so we can print its message
    print(f"An unexpected error occurred: {e}")
```

---

## 4. The `else` Clause
Python allows an optional `else` block. The code inside the `else` block executes **only if the `try` block succeeds without raising any exceptions**. It is highly recommended to keep the `try` block as short as possible and put the success-dependent code in the `else` block.

```python
try:
    data_points = int(input("How many data points did you collect? "))
except ValueError:
    print("Invalid input. Please enter a whole number.")
else:
    # This only runs if the user successfully entered an integer
    print(f"Successfully recorded {data_points} data points.")
    # Proceed with data analysis...
```

---

## 5. The `finally` Clause
The `finally` block is executed **no matter what happens**—whether an exception was raised, whether it was caught, or whether the code ran perfectly. 

**Primary Use Case:** Resource management and cleanup. For example, closing an open file, terminating a database connection, or shutting down a web driver after web scraping, ensuring that memory leaks do not occur even if the script crashes.

```python
try:
    file = open("dataset.csv", "r")
    # Simulate a process that causes an error
    content = file.read()
    data = 10 / 0 
except FileNotFoundError:
    print("Error: The dataset could not be located.")
except ZeroDivisionError:
    print("Error: Division by zero during processing.")
finally:
    # This block will execute regardless of the ZeroDivisionError
    print("Executing cleanup operations...")
    try:
        file.close()
        print("File closed successfully.")
    except NameError:
        pass # File was never opened to begin with
```

---

## 6. Raising Exceptions Customly (`raise`)
Sometimes, Python's logic is fine, but the *business logic* is violated. In these cases, you can intentionally trigger an exception using the `raise` keyword. This is heavily used in validation functions before pushing data to a database.

```python
def validate_age(age):
    if age < 0:
        # We manually raise a ValueError with a custom message
        raise ValueError("Age cannot be a negative number.")
    if age > 120:
        raise ValueError("Age exceeds reasonable human lifespan.")
    return True

try:
    user_age = -5
    validate_age(user_age)
except ValueError as e:
    print(f"Data Validation Failed: {e}")
```

---

## 7. Creating Custom Exceptions
For advanced applications, standard exceptions like `ValueError` might not be descriptive enough. You can create your own exception classes by inheriting from Python's built-in `Exception` base class. This makes your error logs incredibly readable.

```python
# Define a custom exception
class DataPipelineError(Exception):
    """Raised when the data pipeline encounters a critical failure."""
    pass

def load_data(table_name):
    if table_name == "":
        raise DataPipelineError("Table name cannot be empty. Pipeline Halted.")
    print(f"Loading data into {table_name}...")

try:
    load_data("")
except DataPipelineError as e:
    print(f"CRITICAL: {e}")
```
