# Static Methods, Class Methods & Property Decorators

## Overview
In Object-Oriented Programming (OOP) in Python, decorators are a powerful feature used to modify the behavior of functions and methods. The three most essential decorators for advanced class design are `@staticmethod`, `@classmethod`, and `@property`. Understanding these allows you to write cleaner, more encapsulated, and more Pythonic code.

---

## 1. Static Methods (`@staticmethod`)
By default, methods inside a class take the instance (`self`) as their first argument. However, sometimes you have a utility function that logically belongs inside a class because it relates to the class's purpose, but it doesn't actually need to access any instance attributes (`self`) or class attributes (`cls`).

We use the `@staticmethod` decorator for this. It behaves exactly like a regular Python function but is encapsulated within the class's namespace.

### Theoretical Key Points:
* Cannot modify object instance state.
* Cannot modify class state.
* Defined using the `@staticmethod` decorator.
* Does not take `self` or `cls` as the first parameter.

### Code Example
```python
class MathOperations:
    
    @staticmethod
    def add_numbers(x, y):
        """A simple utility method that doesn't need self or cls."""
        return x + y
        
    @staticmethod
    def is_workday(day):
        """Checks if a given date is a weekday."""
        if day.weekday() == 5 or day.weekday() == 6:
            return False
        return True

# Calling static methods directly on the class without instantiating it
sum_result = MathOperations.add_numbers(5, 10)
print(sum_result)  # Output: 15

import datetime
my_date = datetime.date(2026, 4, 21)
print(MathOperations.is_workday(my_date))  # Output: True
```

## 2. Class Methods (`@classmethod`)
While regular methods take the instance (`self`) as the first argument, **class methods** take the class itself as the first argument, conventionally named `cls`. 

Class methods are used **when you need to access or modify class-level variables (which affect all instances) rather than instance-level variables**. They are also widely used to create **"Alternative Constructors"** (methods that return an instance of the class in a different way than the standard `__init__` method).

### Theoretical Key Points:
* Can modify class state (changes apply to all instances of the class).
* Cannot modify specific instance state.
* Defined using the `@classmethod` decorator.
* Takes `cls` as the first parameter.

### Code Example
```python
class Employee:
    raise_amount = 1.04  # Class variable

    def __init__(self, first, last, pay):
        self.first = first
        self.last = last
        self.pay = pay

    @classmethod
    def set_raise_amt(cls, amount):
        """Modifies the class variable for ALL instances."""
        cls.raise_amount = amount

    @classmethod
    def from_string(cls, emp_str):
        """Alternative Constructor: creates an Employee from a hyphen-separated string."""
        first, last, pay = emp_str.split('-')
        return cls(first, last, int(pay))  # Returns a new instance of Employee

# Modifying class state using the class method
Employee.set_raise_amt(1.05)
print(Employee.raise_amount)  # Output: 1.05

# Using the alternative constructor to create an object
new_emp_1 = Employee.from_string("John-Doe-70000")
print(new_emp_1.first)  # Output: John
print(new_emp_1.pay)    # Output: 70000
```

## 3. Property Decorators (`@property`, Setters, and Deleters)
The `@property` decorator allows you to define a method that can be accessed exactly like an attribute. This is crucial for **Encapsulation**. 

Instead of creating explicit `get_value()` and `set_value()` methods (like you would in languages like Java or C++), Python allows you to use property decorators to hide complex logic (like validation or calculations) behind simple attribute access syntax.

### Theoretical Key Points:
* **`@property` (Getter):** Allows a method to be accessed without parentheses `()`. Computes values dynamically based on other attributes.
* **`@method_name.setter` (Setter):** Allows you to assign a value to that property, triggering custom logic or validation when an assignment is made.
* **`@method_name.deleter` (Deleter):** Defines specific behavior/cleanup logic when the `del` keyword is used on the property.

### Code Example
```python
class Employee:
    def __init__(self, first, last):
        self.first = first
        self.last = last

    @property
    def email(self):
        """Accessed as an attribute, computes email dynamically."""
        return f"{self.first.lower()}.{self.last.lower()}@company.com"

    @property
    def fullname(self):
        """Accessed as an attribute."""
        return f"{self.first} {self.last}"

    @fullname.setter
    def fullname(self, name):
        """Triggered when setting: emp.fullname = 'Jane Smith'"""
        first, last = name.split(' ')
        self.first = first
        self.last = last

    @fullname.deleter
    def fullname(self):
        """Triggered when deleting: del emp.fullname"""
        print('Delete Name logic triggered!')
        self.first = None
        self.last = None

# 1. Instantiating the class
emp_1 = Employee("Animesh", "Sanghi")

# 2. Accessing properties (Notice there are no parentheses!)
print(emp_1.fullname)  # Output: Animesh Sanghi
print(emp_1.email)     # Output: animesh.sanghi@company.com

# 3. Using the Setter
emp_1.fullname = "Kiran Sharma"
# The setter updates the underlying self.first and self.last
print(emp_1.first)     # Output: Kiran
print(emp_1.email)     # Output: kiran.sharma@company.com

# 4. Using the Deleter
del emp_1.fullname     # Output: Delete Name logic triggered!
print(emp_1.first)     # Output: None
```

## Summary Comparison

| Decorator | First Argument | Scope of Access | Primary Use Case |
| :--- | :--- | :--- | :--- |
| **None (Regular Method)** | `self` | Instance | Modifying or accessing specific object data. |
| **`@staticmethod`**| `None` | Independent | Utility functions logically related to the class, but needing no class/instance data. |
| **`@classmethod`** | `cls` | Class Level | Modifying class-level variables or creating Alternative Constructors. |
| **`@property`** | `self` | Attribute Style| Encapsulation, dynamic attribute calculation, Custom Getters, Setters, and Deleters. |
