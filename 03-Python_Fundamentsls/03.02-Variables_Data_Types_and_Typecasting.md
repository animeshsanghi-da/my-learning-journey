# Variables, Data Types & Typecasting in Python

## Overview
In Python, handling data efficiently is the first step toward building complex data science pipelines. This module covers how to store data using **Variables**, understand the nature of the data using **Data Types**, and convert data from one form to another using **Typecasting**.

---

## 1. Variables
A **Variable** is essentially a named container in your computer's memory used to store data. Unlike languages such as C++ or Java, Python is dynamically typed, meaning you do not need to declare a variable's type before assigning a value to it.

### Variable Assignment
You assign a value to a variable using the assignment operator `=`.

```python
a = 30        # Assigns the integer 30 to variable 'a'
b = "Animesh" # Assigns the string "Animesh" to variable 'b'
c = 71.22     # Assigns the floating-point number 71.22 to variable 'c'
```

### Rules for Naming Variables
To ensure your code executes without syntax errors, you must follow Python's rules for naming variables (identifiers):
1. A variable name can contain **alphabets, digits, and underscores** (`_`).
2. A variable name **cannot** start with a digit. (e.g., `1name` is invalid, but `name1` is valid).
3. A variable name **cannot** contain white spaces. (Use underscores instead: `first_name`).
4. Special characters (like `@`, `!`, `#`, `$`, `%`) are not allowed inside variable names.
5. You cannot use Python **keywords** (reserved words like `if`, `else`, `class`, `def`) as variable names.

## 2. Data Types
A Data Type specifies the type of value a variable holds. This dictates what operations can be performed on that data. Python has several built-in primary [data types](../linked_files/data_types.py):

![](../Images/data_types_in_python.png)
* **Integers (`int`):** Whole numbers without a fractional part (e.g., `1`, `-5`, `1000`).
* **Floating-Point Numbers (`float`):** Numbers that contain a decimal point (e.g., `3.14`, `-0.001`, `2.0`).
* **Strings (`str`):** A sequence of characters enclosed in single (`' '`), double (`" "`), or triple (`''' '''`) quotes (e.g., `"Hello, World!"`, `'Animesh'`).
* **Booleans (`bool`):** Represents truth values. Can only be `True` or `False` (Note: the first letter must be capitalized).
* **NoneType (`None`):** Represents the intentional absence of a value. It is used to define a null variable or object.

### The `type()` Function
To find out what type of data a variable is storing, you can use the built-in `type()` function.

```python
name = "Animesh"
age = 29
is_student = True
salary = None

print(type(name))        # Output: <class 'str'>
print(type(age))         # Output: <class 'int'>
print(type(is_student))  # Output: <class 'bool'>
print(type(salary))      # Output: <class 'NoneType'>
```

## 3. Typecasting (Type Conversion)
**Typecasting** is the process of converting a variable from one data type into another. This is particularly useful when taking user inputs (which are always strings by default) and converting them into numbers for mathematical operations.

### Explicit Typecasting
Explicit typecasting is when the programmer manually converts the data type using built-in Python functions such as `int()`, `float()`, or `str()`.

```python
# Converting a string to an integer
num_str = "31"
num_int = int(num_str)
print(type(num_int))  # Output: <class 'int'>

# Converting an integer to a float
a = 10
b = float(a)
print(b)              # Output: 10.0

# Converting a float to a string
c = 3.14
d = str(c)
print(type(d))        # Output: <class 'str'>
```

*Note on Explicit Casting:* You can only cast a string to an integer if the string contains a valid numeric literal. Trying `int("Animesh")` will throw a `ValueError`.

### Implicit Typecasting
Implicit typecasting occurs automatically when Python temporarily converts a smaller or less complex data type into a larger or more complex data type to prevent data loss during operations.

```python
a = 5        # Integer
b = 2.5      # Float

# Python automatically converts 'a' to a float before adding
result = a + b  

print(result)        # Output: 7.5
print(type(result))  # Output: <class 'float'>
```

## 4. Practical Example
Here is a small script demonstrating how variables, data types, and typecasting come together in a real-world scenario:

```python
# Simulating fetching user data from a web form (always comes as strings)
user_age_input = "29"
account_balance_input = "1500.50"

# Checking initial types
print(type(user_age_input))  # <class 'str'>

# Typecasting to appropriate mathematical types
age = int(user_age_input)
balance = float(account_balance_input)

# Performing operations
future_age = age + 5
bonus = balance * 0.10

print(f"In 5 years, the user will be {future_age} years old.")
print(f"The user's 10% bonus is ${bonus}.")
```
