# Pandas Data Cleaning: Missing Values & Duplicates

## Overview
Real-world data is rarely perfect. It often comes with missing entries, corrupted records, and duplicated rows. In the Data Analytics Lifecycle, this falls under the **Process** phase (Data Cleaning). The `pandas` library in Python provides highly efficient, vectorized methods for detecting, filtering, and resolving these issues to prepare a pristine dataset for exploratory data analysis (EDA) and machine learning.

---

## 1. Handling Missing Data (NaN)
In Pandas, missing data is typically represented by the floating-point value `NaN` (Not a Number) or `None`. Handling these correctly is critical because machine learning models and statistical formulas will throw errors or return skewed results if fed `NaN` values.

### A. Detecting Missing Values
Before fixing missing data, you must know where it exists and how much of it there is.

```python
import pandas as pd
import numpy as np

# Sample DataFrame with missing values
data = {
    'Name': ['Aarav', 'Sneha', 'Raj', 'Fatima', np.nan],
    'Age': [25, np.nan, 28, 22, 29],
    'Salary': [60000, 75000, np.nan, 85000, 70000]
}
df = pd.DataFrame(data)

# Check for missing values (returns a boolean DataFrame)
print(df.isnull())

# Count missing values per column (Highly Recommended)
print(df.isnull().sum())

# Total missing values in the entire DataFrame
print(df.isnull().sum().sum())
```

### B. Dropping Missing Data (`dropna`)
If the dataset is large and the missing values are negligible, the simplest approach is to remove the rows or columns containing them.

```python
# Drop any row that contains at least one NaN
df_cleaned_rows = df.dropna(axis=0)

# Drop any column that contains at least one NaN
df_cleaned_cols = df.dropna(axis=1)

# Drop rows only if ALL values in that row are missing
df.dropna(how='all', inplace=True)

# Drop rows only if there is a missing value in a SPECIFIC column
df.dropna(subset=['Age', 'Salary'], inplace=True)

# Keep rows that have at least a minimum number of non-NaN values
df.dropna(thresh=2, inplace=True)
```

### C. Imputing/Filling Missing Data (`fillna`)
Dropping data means losing information. Often, it is better to "impute" or fill the missing values with a logical substitute (like the mean, median, mode, or a static value).

```python
# Fill all NaN values in the DataFrame with a specific static value (e.g., 0)
df.fillna(0)

# Fill missing categorical values with a specific string
df['Name'].fillna('Unknown', inplace=True)

# Impute numerical columns with statistical measures (Mean/Median/Mode)
# Using median is often safer if there are extreme outliers in the column
mean_age = df['Age'].mean()
df['Age'].fillna(mean_age, inplace=True)

# Forward Fill (propagate the previous valid observation forward) - Great for Time Series
df.fillna(method='ffill', inplace=True)

# Backward Fill (use the next valid observation to fill the gap)
df.fillna(method='bfill', inplace=True)
```

## 2. Handling Duplicates
Duplicate rows can artificially inflate certain statistics, leading to biased machine learning models and inaccurate business reports. 

### A. Detecting Duplicates
The `duplicated()` method returns a boolean Series denoting whether a row is a duplicate of a preceding row.

```python
# Sample DataFrame with duplicates
data_dup = {
    'TransactionID': [101, 102, 103, 102, 104, 101],
    'Amount': [250, 400, 150, 400, 300, 250]
}
df_dup = pd.DataFrame(data_dup)

# Identify which rows are duplicates (marks the second occurrence as True)
print(df_dup.duplicated())

# Count the total number of duplicate rows
print(df_dup.duplicated().sum())

# View the actual duplicate rows
print(df_dup[df_dup.duplicated()])
```

### B. Removing Duplicates (`drop_duplicates`)
Once identified, you can remove these redundant rows.

```python
# Drop duplicate rows, keeping the first occurrence (default behavior)
df_dup.drop_duplicates(inplace=True)

# Drop duplicate rows, keeping the LAST occurrence instead
df_dup.drop_duplicates(keep='last', inplace=True)

# Drop all duplicates entirely (if a row appears more than once, remove all instances of it)
df_dup.drop_duplicates(keep=False, inplace=True)

# Drop duplicates based on a specific column subset (e.g., duplicate Transaction IDs only)
df_dup.drop_duplicates(subset=['TransactionID'], inplace=True)
```

---

## 3. Best Practices for Data Cleaning
1. **Always Make a Copy:** Before applying destructive operations, copy your dataframe: `df_clean = df.copy()`.
2. **Investigate the "Why":** Don't just blindly drop `NaN` values. Investigate *why* they are missing. Is it a system error? Did a user skip a form field? The absence of data can sometimes be a signal in itself.
3. **Imputation Strategy:** Use `mean` for normally distributed data, `median` for skewed data, and `mode` (most frequent) for categorical data.
4. **Permanent Changes:** Remember to use `inplace=True` or reassign the variable (`df = df.dropna()`) to make your changes persist in memory.
