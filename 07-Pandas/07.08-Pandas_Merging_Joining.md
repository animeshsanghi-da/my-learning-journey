# Pandas Data Manipulation: Merging, Joining & Concatenating

## Overview
In real-world data science, information is rarely contained in a single table. Data is often normalized and split across multiple files or databases (e.g., customer details in one table, and their purchase history in another). Pandas provides powerful, high-performance methods to combine these datasets, heavily inspired by SQL relational databases.

This section covers the three primary ways to combine DataFrames in Pandas: **Concatenating**, **Merging**, and **Joining**.

---

## 1. Concatenation (`pd.concat`)
Concatenation is the process of "gluing" or "stacking" DataFrames together either vertically (row-wise) or horizontally (column-wise).

### Vertical Concatenation (Stacking Rows)
Used when you have two DataFrames with the exact same columns (e.g., sales data for January and sales data for February) and you want to stack them on top of each other.

```python
import pandas as pd

# Creating sample DataFrames
df_jan = pd.DataFrame({'ID': [1, 2], 'Sales': [200, 250]})
df_feb = pd.DataFrame({'ID': [3, 4], 'Sales': [300, 150]})

# Stacking vertically (default axis=0)
# ignore_index=True ensures the new DataFrame has a continuous index from 0 to n
df_combined = pd.concat([df_jan, df_feb], ignore_index=True)

print(df_combined)
# Output:
#    ID  Sales
# 0   1    200
# 1   2    250
# 2   3    300
# 3   4    150
```

### Horizontal Concatenation (Stacking Columns)
Used when you want to glue DataFrames side-by-side. Ensure that the row indices match up correctly.

```python
# Stacking horizontally
df_combined_h = pd.concat([df_jan, df_feb], axis=1)
```

## 2. Merging (`pd.merge`)
Merging is the most powerful method for combining DataFrames. It is the Pandas equivalent of **SQL JOINs**. It combines DataFrames based on the values of shared columns (called "keys").

### Types of Merges (The `how` parameter)
1. **Inner Merge (Default):** Keeps only the rows where the key exists in *both* DataFrames.
2. **Left Merge:** Keeps all rows from the Left DataFrame, and matches records from the Right DataFrame. Fills missing values with `NaN`.
3. **Right Merge:** Keeps all rows from the Right DataFrame, and matches records from the Left.
4. **Outer Merge:** Keeps *all* rows from both DataFrames. Fills missing matches with `NaN`.

### Example: The Merging Syntax
```python
# Sample Data
users = pd.DataFrame({
    'User_ID': [1, 2, 3, 4],
    'Name': ['Alice', 'Bob', 'Charlie', 'David']
})

transactions = pd.DataFrame({
    'Transaction_ID': [101, 102, 103],
    'User_ID': [1, 2, 1], # Notice User 3 and 4 have no transactions
    'Amount': [250, 400, 150]
})

# 1. INNER MERGE (Only users who made a transaction)
inner_merged = pd.merge(users, transactions, on='User_ID', how='inner')
print(inner_merged)

# 2. LEFT MERGE (All users, even if they didn't buy anything)
left_merged = pd.merge(users, transactions, on='User_ID', how='left')
print(left_merged)
# Charlie and David will have 'NaN' in their Transaction_ID and Amount columns.

# 3. MERGING ON DIFFERENT COLUMN NAMES
# If the keys have different column names in each DataFrame
# e.g., 'User_ID' in df1 and 'ID' in df2:
# merged = pd.merge(df1, df2, left_on='User_ID', right_on='ID', how='inner')
```

## 3. Joining (`df.join`)
While `pd.merge()` is optimized for joining on *columns*, `df.join()` is a convenient method optimized for joining DataFrames on their **Indices** (row labels).

By default, `.join()` performs a **Left Join**.

```python
# Setting indices
df_a = pd.DataFrame({'A': ['A0', 'A1', 'A2']}, index=['K0', 'K1', 'K2'])
df_b = pd.DataFrame({'B': ['B0', 'B1', 'B2']}, index=['K0', 'K2', 'K3'])

# Joining based on index
joined_df = df_a.join(df_b)

print(joined_df)
# Output:
#      A    B
# K0  A0   B0
# K1  A1  NaN
# K2  A2   B1
```

---

## Summary Cheat Sheet
* Need to stack DataFrames physically? Use **`pd.concat()`**
* Need to perform SQL-like relationships using shared columns? Use **`pd.merge()`**
* Need to combine DataFrames using their Row Indices? Use **`df.join()`**
