# Pandas: Series vs. DataFrames

## Overview
**Pandas** is the most popular Python library for data manipulation and analysis. Built on top of NumPy, it provides fast, flexible, and expressive data structures designed to make working with "relational" or "labeled" data both easy and intuitive. It is the cornerstone of the **Process** and **Analyze** phases in Data Science.

The name "pandas" is derived from the econometric term **"Panel Data"** (multidimensional data) and **"Python Data Analysis"**.

To master Pandas, you must first understand its two primary data structures: the **Series** (1-Dimensional) and the **DataFrame** (2-Dimensional).

---

## 1. Pandas Series
A **Series** is a one-dimensional labeled array capable of holding data of any type (integer, string, float, python objects, etc.). 

Think of it as a single column in an Excel spreadsheet. The axis labels are collectively called the **index**.

### Creating a Series
You can create a Series from a Python list, dictionary, or a NumPy array.

```python
import pandas as pd
import numpy as np

# 1. Creating a Series from a list (default integer index)
data_list = [10, 20, 30, 40]
s1 = pd.Series(data_list)
print("Series from List:\n", s1)

# 2. Creating a Series with a custom index
s2 = pd.Series([10, 20, 30], index=['A', 'B', 'C'])
print("\nSeries with Custom Index:\n", s2)

# 3. Creating a Series from a Dictionary (keys become index)
data_dict = {'Math': 90, 'Science': 85, 'English': 88}
s3 = pd.Series(data_dict)
print("\nSeries from Dictionary:\n", s3)
```

### Key Attributes of a Series
* `.values`: Returns the data as a NumPy array.
* `.index`: Returns the index (labels) of the Series.
* `.dtype`: Returns the data type of the elements.

---

## 2. Pandas DataFrames
A **DataFrame** is a two-dimensional, size-mutable, potentially heterogeneous tabular data structure with labeled axes (rows and columns). 

Think of a DataFrame as an entire Excel spreadsheet or a SQL table. It is essentially a dictionary of Series objects, where each Series represents a column, and they all share the same index.

### Creating a DataFrame
DataFrames are most commonly created by importing files (like CSVs), but they can also be created from scratch using Python dictionaries or lists of lists.

```python
# 1. Creating a DataFrame from a Dictionary of Lists
# (Keys become column names, lists become the column data)
data = {
    'Name': ['Aarav', 'Sneha', 'Raj', 'Fatima'],
    'Age': [25, 28, 22, 30],
    'City': ['Mumbai', 'Kolkata', 'Delhi', 'Chennai']
}
df1 = pd.DataFrame(data)
print("DataFrame from Dict:\n", df1)

# 2. Creating a DataFrame from a List of Dictionaries
# (Each dictionary represents a row)
row_data = [
    {'Name': 'Aarav', 'Salary': 80000},
    {'Name': 'Sneha', 'Salary': 75000},
    {'Name': 'Raj', 'Salary': 72000}
]
df2 = pd.DataFrame(row_data)
print("\nDataFrame from List of Dicts:\n", df2)
```

### Key Attributes of a DataFrame
* `.columns`: Returns the column labels.
* `.index`: Returns the row labels (index).
* `.shape`: Returns a tuple representing the dimensionality (rows, columns).
* `.dtypes`: Returns the data types of each column.

---

## 3. Series vs. DataFrame: Summary Comparison

| Feature | Series | DataFrame |
| :--- | :--- | :--- |
| **Dimensions** | 1-Dimensional (1D) | 2-Dimensional (2D) |
| **Structure** | A single column of data with an index | A table with rows and columns (Multiple Series) |
| **Data Types** | Homogeneous (usually holds one data type) | Heterogeneous (different columns can have different data types) |
| **Size Mutable** | Size can be changed (though less common) | Size is mutable (easy to add/drop rows and columns) |

## 4. Extracting a Series from a DataFrame
When you select a single column from a DataFrame, Pandas automatically returns it as a Series.

```python
# Extracting the 'Age' column from df1
age_series = df1['Age']

print(type(age_series)) 
# Output: <class 'pandas.core.series.Series'>

print(age_series)
# Output:
# 0    25
# 1    28
# 2    22
# 3    30
# Name: Age, dtype: int64
```
