# Pandas: Creating & Exploring DataFrames (EDA)

## Overview
In the Data Science lifecycle, **Exploratory Data Analysis (EDA)** is the crucial first step after acquiring data. It involves investigating the dataset to discover patterns, spot anomalies, test hypotheses, and check assumptions. **Pandas** DataFrames are the industry standard for performing EDA in Python.

A DataFrame is a 2-dimensional labeled data structure with columns of potentially different types. You can think of it like a spreadsheet or SQL table, or a dictionary of Series objects.

---

## 1. Creating DataFrames
While data is usually loaded from external files, you can also create DataFrames from scratch using Python data structures.

### From a Dictionary
The most common way to create a DataFrame from scratch is using a dictionary where keys become column headers and values (lists) become the column data.

```python
import pandas as pd

data = {
    'Name': ['Alice', 'Bob', 'Charlie', 'David'],
    'Age': [25, 30, 35, 40],
    'City': ['New York', 'Paris', 'London', 'Tokyo'],
    'Salary': [70000, 80000, 120000, 90000]
}

df = pd.DataFrame(data)
print(df)
```

### From a List of Lists
You can also pass a list of lists and specify the column names separately.

```python
data_lists = [
    ['Alice', 25, 'New York'],
    ['Bob', 30, 'Paris']
]
df_lists = pd.DataFrame(data_lists, columns=['Name', 'Age', 'City'])
```

### Loading from External Files (The Standard Workflow)
In real-world data analytics, you will read data from files like CSVs, Excel, or JSON.

```python
# Loading a CSV file
df_csv = pd.read_csv('dataset.csv')

# Loading an Excel file
df_excel = pd.read_excel('dataset.xlsx', sheet_name='Sheet1')
```

## 2. Exploring the DataFrame (EDA Basics)
Once your data is loaded, the immediate next step is to understand its structure, size, and statistical properties.

### Viewing the Data
Instead of printing the entire dataset (which could be millions of rows), you can inspect small samples.

```python
# View the first 5 rows (default is 5)
print(df.head())

# View the last 3 rows
print(df.tail(3))

# View a random sample of 5 rows (Great for checking randomization)
print(df.sample(5))
```

### Inspecting Structure and Metadata
Understanding the dimensions and data types is essential before any manipulation.

```python
# Get the dimensions (rows, columns) - Returns a tuple
print(df.shape)  

# List all column names
print(df.columns)

# Check the data type of each column (e.g., int64, float64, object)
print(df.dtypes)

# Get a concise summary of the DataFrame (Includes non-null counts and memory usage)
df.info()
```

### Statistical Summary
The `describe()` method computes a summary of statistics pertaining to the DataFrame columns.

```python
# For numerical columns (count, mean, std, min, 25%, 50%, 75%, max)
print(df.describe())

# For categorical/object columns (count, unique, top, freq)
print(df.describe(include=['object']))
```

### Exploring Categorical Variables
When dealing with categorical data (like 'City' or 'Gender'), you often want to know the frequency of each category.

```python
# Count the occurrences of each unique value in a specific column
print(df['City'].value_counts())

# Get the frequencies as percentages
print(df['City'].value_counts(normalize=True) * 100)

# Get an array of the unique values themselves
print(df['City'].unique())

# Get the number of unique values
print(df['City'].nunique())
```

---

## Summary
The combination of `.info()`, `.describe()`, and `.head()` forms the holy trinity of the initial EDA phase. Before attempting to clean data, build machine learning pipelines, or create Tableau/Power BI visualizations, these pandas commands ensure you thoroughly understand the dataset's architecture and potential flaws.
