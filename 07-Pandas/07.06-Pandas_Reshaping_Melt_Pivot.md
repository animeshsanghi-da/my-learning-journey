# Pandas: Reshaping Data (Melt & Pivot)

## Overview
Data rarely comes in the exact format you need for analysis or machine learning. In data analytics, you often need to transform datasets between **Wide Format** (where each variable has its own column) and **Long Format** (where variables are collapsed into key-value pairs). 

Pandas provides powerful tools to reshape DataFrames. The two most fundamental operations for this are **Melt** (unpivoting) and **Pivot** (creating spreadsheet-style pivot tables). Mastering these is crucial for preparing data for visualization libraries like Seaborn, which heavily favor long-format data.

---

## 1. Melting Data: Wide to Long (`pd.melt()`)

The `pd.melt()` function "unpivots" a DataFrame from wide format to long format. It leaves specified identifier columns intact while gathering all other columns into two new columns: a "variable" column and a "value" column.

### Key Parameters:
* **`id_vars`**: Columns to keep as identifiers (these won't be melted).
* **`value_vars`**: Specific columns to unpivot (if left blank, Pandas melts all columns not in `id_vars`).
* **`var_name`**: The name to give to the new "variable" column (defaults to "variable").
* **`value_name`**: The name to give to the new "value" column (defaults to "value").

### Example: Unpivoting Sales Data
```python
import pandas as pd

# Creating a Wide-format DataFrame
data = {
    'Region': ['North', 'South', 'East', 'West'],
    '2024_Q1_Sales': [1500, 2200, 1800, 2500],
    '2024_Q2_Sales': [1600, 2100, 1900, 2600]
}
df_wide = pd.DataFrame(data)

# Melting the DataFrame
df_long = pd.melt(
    df_wide, 
    id_vars=['Region'], 
    value_vars=['2024_Q1_Sales', '2024_Q2_Sales'],
    var_name='Quarter', 
    value_name='Sales'
)

print(df_long)
# Output will have columns: [Region, Quarter, Sales]
# with 8 rows instead of 4.
```

## 2. Pivoting Data: Long to Wide (`df.pivot()`)

The `pivot()` method does the exact opposite of `melt()`. It takes long-format data and turns it back into wide format by distributing the values of one column across multiple new columns. 

*Note: `pivot()` does not support data aggregation. If you have duplicate entries for a given index/column combination, `pivot()` will throw a `ValueError`.*

### Key Parameters:
* **`index`**: The column whose unique values will become the new rows (the index).
* **`columns`**: The column whose unique values will become the new column headers.
* **`values`**: The column whose values will populate the new DataFrame cells.

### Example: Reversing the Melt
```python
# Pivoting the previously melted df_long back to wide format
df_pivoted = df_long.pivot(
    index='Region', 
    columns='Quarter', 
    values='Sales'
)

# Optional: Resetting the index to flatten the DataFrame structure
df_pivoted = df_pivoted.reset_index()

print(df_pivoted)
```

## 3. Pivot Tables with Aggregation (`pd.pivot_table()`)

When your data contains multiple records for the same index/column combination (e.g., multiple sales entries for the "North" region in "Q1"), `pivot()` will fail. In this scenario, you must use `pd.pivot_table()`, which mathematically aggregates the duplicate values.

This functions exactly like a Pivot Table in Excel or Google Sheets.

### Key Parameters:
* **`index`**: Column(s) to group by on the rows.
* **`columns`**: Column(s) to group by on the columns.
* **`values`**: Column(s) to aggregate.
* **`aggfunc`**: The mathematical function to apply to the grouped data (e.g., `'mean'`, `'sum'`, `'count'`, `'max'`). The default is `'mean'`.
* **`margins`**: Set to `True` to add row/column subtotals and grand totals (equivalent to ROLLUP in SQL).

### Example: Summarizing Complex Data
```python
import numpy as np

# Creating a complex dataset with duplicate index/column pairs
raw_data = {
    'Store': ['Store_A', 'Store_A', 'Store_A', 'Store_B', 'Store_B', 'Store_B'],
    'Product_Category': ['Electronics', 'Electronics', 'Furniture', 'Electronics', 'Furniture', 'Furniture'],
    'Revenue': [5000, 6000, 1500, 8000, 2000, 2500]
}
df_sales = pd.DataFrame(raw_data)

# Creating a Pivot Table to sum the revenue
pivot_df = pd.pivot_table(
    df_sales,
    index='Store',
    columns='Product_Category',
    values='Revenue',
    aggfunc='sum',  # We can pass numpy functions like np.sum or string aliases
    margins=True,   # Adds an "All" column/row for totals
    fill_value=0    # Replaces any NaN values with 0
)

print(pivot_df)
```

---

## 4. When to Use Which?
* **Use `melt()`** when you need to flatten datasets for visualizations (like Seaborn's `catplot` or `relplot`) or for feeding into Scikit-learn machine learning pipelines.
* **Use `pivot()`** when you want to re-organize distinct, non-overlapping time-series data or categorical data for easier reading.
* **Use `pivot_table()`** when you need to calculate business metrics (sums, averages, counts) across multidimensional grouped data, much like traditional Business Intelligence workflows.
