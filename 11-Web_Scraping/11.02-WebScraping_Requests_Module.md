# Web Scraping: Fetching Data with the Requests Module

## Overview
Web scraping is the process of automatically extracting data from websites. The first step in any scraping pipeline is fetching the raw HTML content of a webpage. In Python, the standard library for this task is `requests`. It allows you to send HTTP requests to a server and receive the data it sends back.

---

## 1. What is an HTTP Request?
When you visit a URL in your browser, you are sending a "GET" request to a server. The server then responds with data (usually HTML, CSS, and JavaScript). For web scraping, we use Python to mimic this behavior.

### Common HTTP Methods:
* **GET**: Used to retrieve data from a server (most common for scraping).
* **POST**: Used to send data to a server (e.g., submitting a form or logging in).

---

## 2. Setting Up the Environment
To use the module, you must first install it using `pip` and then import it into your script.

```python
# Install the library via terminal
# pip install requests

import requests
```

---

## 3. Basic GET Request
The `requests.get()` function is used to fetch a page. You can then check the "Status Code" to see if the request was successful.

```python
url = "https://www.example.com"
response = requests.get(url)

# Status Code 200 means success
print(f"Status Code: {response.status_code}")

# Extracting the raw HTML content
html_content = response.text
print(html_content[:500])  # Printing the first 500 characters
```

### Common Status Codes:
* **200**: OK (Success).
* **404**: Not Found (The URL does not exist).
* **403**: Forbidden (The server is blocking your request).
* **500**: Internal Server Error.

---

## 4. Handling Headers and User-Agents
Many websites block automated scripts to prevent server overload. To bypass basic blocks, you can provide a `User-Agent` header to make your script look like a real web browser.

```python
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36"
}

response = requests.get("https://www.amazon.com", headers=headers)
print(response.status_code)
```

---

## 5. Working with Parameters
If you are scraping a search result page, you often need to pass parameters into the URL (e.g., `?q=python`). `requests` handles this using a dictionary.

```python
query_params = {
    "q": "Data Science",
    "region": "India"
}

response = requests.get("https://www.google.com/search", params=query_params)
# This effectively requests: https://www.google.com/search?q=Data+Science&region=India
```

---

## 6. Saving Content to a File
Once you have fetched the HTML, it is best practice to save it locally before moving to the parsing stage (using BeautifulSoup). This prevents you from making unnecessary repeated requests to the server.

```python
url = "https://www.codewithharry.com"
r = requests.get(url)

with open("harry_site.html", "w", encoding="utf-8") as f:
    f.write(r.text)
```

---

## 7. Error Handling
When scraping, connections can fail. Using `try-except` blocks ensures your script doesn't crash.

```python
try:
    response = requests.get("https://invalid-url-example.com", timeout=5)
    response.raise_for_status() # Raises an error for 4xx or 5xx codes
except requests.exceptions.HTTPError as errh:
    print("Http Error:", errh)
except requests.exceptions.ConnectionError as errc:
    print("Error Connecting:", errc)
except requests.exceptions.Timeout as errt:
    print("Timeout Error:", errt)
except requests.exceptions.RequestException as err:
    print("Something went wrong", err)
```
