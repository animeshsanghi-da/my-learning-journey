# Git Stash: Managing Temporary Changes

## Overview
Git stashing is a powerful feature that allows you to temporarily shelve (or "stash") changes you've made to your working directory so you can work on something else, and then come back and re-apply them later. This is particularly useful when you need to quickly switch context—such as jumping from a feature branch to fix a critical bug on the main branch—without being forced to make an unnecessary or "junk" commit.

---

## 1. What is Git Stash?
When you stash, Git takes the uncommitted changes (both staged and unstaged) and saves them in an internal stack. Your working directory is then returned to a clean state matching the `HEAD` commit.

### When to use Stashing:
* **Context Switching:** You are in the middle of a feature and need to switch branches immediately.
* **Clean Pulls:** You want to pull the latest changes from a remote repository but have local work that isn't ready for a commit.
* **Experimentation:** You want to try a new coding approach but want a "save point" to return to if it fails.

---

## 2. Basic Stash Operations

### Saving Changes
To stash your current changes, use:
```bash
git stash
```
*Note: By default, this only stashes tracked files. To include new (untracked) files, use:*
```bash
git stash -u
```

### Listing Stashes
You can view all the items in your stash stack:
```bash
git stash list
```
Output example:
`stash@{0}: On main: work in progress on feature X`
`stash@{1}: WIP on develop: 5002d47 fix conflict`

---

## 3. Re-applying Stashed Changes

Once you are ready to continue your work, you have two primary ways to bring back your shelved changes:

### Apply (Keep in Stash)
This applies the changes but keeps them saved in the stash list.
```bash
git stash apply
```
*To apply a specific stash from the list:*
```bash
git stash apply stash@{2}
```

### Pop (Apply and Remove)
This is the most common method. It applies the most recent stash and immediately removes it from the stash stack.
```bash
git stash pop
```

---

## 4. Managing and Removing Stashes

### View Stash Contents
To see what exactly is inside a stash without applying it:
```bash
git stash show      # Shows file summary
git stash show -p   # Shows the full diff (code changes)
```

### Deleting Stashes
```bash
git stash drop stash@{1} # Remove a specific stash
git stash clear          # Remove ALL stashes permanently
```

---

## 5. Advanced Stashing Techniques

### Creating a Branch from a Stash
If the changes in your stash now conflict with the updated work on your current branch, you can create a new branch specifically for those stashed changes:
```bash
git stash branch new-feature-branch
```

### Stashing Specific Files (Interactive)
If you only want to stash some of your changes and commit others, use the interactive mode:
```bash
git stash -p
```
*Git will go through each "hunk" of code and ask if you want to stash it.*

---

## 6. Summary Table of Stash Commands

| Command | Description |
| :--- | :--- |
| `git stash` | Save tracked changes to the stack. |
| `git stash -u` | Save tracked and untracked files to the stack. |
| `git stash list` | View all stashed changes. |
| `git stash apply` | Re-apply changes without deleting from the stack. |
| `git stash pop` | Re-apply changes and delete from the stack. |
| `git stash show -p` | View the detailed code diff in the latest stash. |
| `git stash drop` | Delete a specific stash entry. |
| `git stash clear` | Wipe the entire stash history. |
