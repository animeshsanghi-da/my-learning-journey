# Git Basics: Foundations of Version Control

## Overview
Git is a Distributed Version Control System (DVCS) that allows developers and data analysts to track changes in their code, collaborate with others, and revert to previous versions of a project if something goes wrong. In the Data Science lifecycle, Git ensures that your analysis is reproducible and your codebase is organized.

---

## 1. Core Concepts
Before running commands, it is essential to understand the three main areas of a Git project:
* **Working Directory:** The actual files you are currently editing on your computer.
* **Staging Area (Index):** A "preview" area where you gather changes that you want to include in your next commit.
* **Repository (.git folder):** The permanent record of your project's history, where Git stores all your snapshots (commits).

---

## 2. Setting Up Git
Before you start tracking projects, you need to configure your identity.

```bash
# Set your name and email
git config --global user.name "Animesh Sanghi"
git config --global user.email "animeshsanghi.da@gmail.com"

# Check your configuration
git config --list
```

---

## 3. Initializing and Tracking Changes
To start tracking a new project or an existing folder, follow these fundamental steps:

### Initialize a Repository
Navigate to your project folder in the terminal and run:
```bash
git init
```
This creates a hidden `.git` folder that tracks all future changes.

### Checking Status
Always check what Git is seeing in your directory:
```bash
git status
```

### Adding Files to the Staging Area
You must tell Git which changes you want to save.
```bash
# Add a specific file
git add filename.py

# Add all changed and new files
git add .
```

### Committing Changes
A commit is a "snapshot" of your project at a specific point in time. Always include a descriptive message.
```bash
git commit -m "Initial commit: Add data cleaning script and raw dataset"
```

---

## 4. Viewing History
Git allows you to look back at every change made to the project.

### The Commit Log
```bash
# View full history
git log

# View a simplified, one-line version of the history
git log --oneline

# View history with a graph of branches
git log --graph --oneline --all
```

---

## 5. Reverting and Undoing
One of Git's greatest strengths is the ability to fix mistakes.

### Unstaging a File
If you added a file to the staging area by mistake:
```bash
git restore --staged filename.py
```

### Discarding Changes in Working Directory
To revert a file back to its state at the last commit:
```bash
git restore filename.py
```

### Amending the Last Commit
If you forgot to add a file or made a typo in your commit message:
```bash
git commit --amend -m "Updated commit message"
```

---

## 6. Ignoring Files with .gitignore
In Data Science, we often have large datasets (CSVs, JSONs) or environment files (like `.env`) that shouldn't be uploaded to GitHub. We use a `.gitignore` file to tell Git to ignore them.

**Example .gitignore content:**
```text
# Ignore python virtual environments
venv/
__pycache__/

# Ignore large datasets
data/raw/
*.csv

# Ignore environment variables
.env
```
