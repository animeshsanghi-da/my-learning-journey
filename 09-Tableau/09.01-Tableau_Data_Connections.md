# Tableau Public: Connecting to Data Sources & Data Blending

## Overview
Before you can build interactive dashboards and uncover insights in Tableau Public, you must first connect to your data. Tableau's robust data engine allows you to connect to various file types, combine multiple tables, and structure your data model efficiently for analysis.

---

## 1. Supported Data Sources in Tableau Public
Tableau Public is a free version of Tableau tailored for public data visualization. While it does not support direct connections to enterprise databases (like MySQL or PostgreSQL), it natively supports a wide array of flat files:
* **Microsoft Excel** (`.xls`, `.xlsx`)
* **Text Files** (`.csv`, `.txt`, `.tsv`)
* **JSON Files**
* **Spatial Files** (Shapefiles, GeoJSON, KML)
* **Statistical Files** (SAS, SPSS, R)
* **Google Sheets** (Cloud connection)
* **OData** (Web data feeds)

*Note: Tableau Public saves all visualizations to the public cloud. It automatically creates a Data Extract (a compressed snapshot of your data) rather than maintaining a Live database connection.*

---

## 2. The Data Source Page
When you load a file into Tableau, you are taken to the **Data Source Page**. This is your canvas for data modeling.
* **Left Pane (Connections):** Shows your connected files and the sheets/tables within them.
* **Canvas:** The area where you drag and drop tables to build your data model.
* **Data Grid:** A preview of your actual data rows and columns.
* **Metadata Grid:** Shows the data types (String, Date, Number, Boolean, Geographic Role) and allows you to rename columns or change data types before loading them into a worksheet.

---

## 3. Combining Data: Relationships vs. Joins

When you need to analyze data from multiple tables (e.g., a `Sales` table and a `Customers` table), Tableau offers two distinct layers for combining data: The Logical Layer and the Physical Layer.

### A. Relationships (The Logical Layer)
Introduced in recent versions of Tableau, **Relationships** (often called "Noodles") are the default and recommended way to combine data. 
* **How it works:** You drag tables onto the canvas, and Tableau draws a flexible "noodle" between them. You define the matching fields (e.g., `Customer_ID = Customer_ID`).
* **Why use it?** Relationships do not merge the tables into a single flat file. They keep the tables separate and only query them at the appropriate level of detail (granularity) when you drag fields onto your dashboard. This prevents data duplication and skewed aggregations.

### B. Joins (The Physical Layer)
If you double-click a logical table on the canvas, you enter the Physical Layer, where you can perform traditional SQL-style Joins. This physically merges the tables into a single wide dataset before analysis.
* **Inner Join:** Returns only records with matching keys in both tables.
* **Left Join:** Returns all records from the left table, and matched records from the right table. (Unmatched right records return NULL).
* **Right Join:** Returns all records from the right table, and matched records from the left table.
* **Full Outer Join:** Returns all records from both tables, filling in NULLs where there are no matches.

```sql
-- Conceptual equivalence of a Tableau Left Join
SELECT a.*, b.*
FROM Orders a
LEFT JOIN Customers b
ON a.Customer_ID = b.Customer_ID;
```

## 4. Unioning Data
While Joins and Relationships combine tables horizontally (adding more columns), **Unions** combine tables vertically (stacking rows on top of each other).
* **Use Case:** You have separate Excel sheets for `2024_Sales` and `2025_Sales`. Both sheets have the exact same column headers.
* **How to do it:** In the Data Source page, drag the first sheet to the canvas, then drag the second sheet *directly underneath* the first sheet until you see the "Drag table to Union" prompt.
* **Tableau Feature:** Tableau automatically generates a `Table Name` and `Sheet Name` column so you can track the origin of each row.

## 5. Data Blending
Data Blending is a unique Tableau feature used to combine data from entirely different data sources (e.g., an Excel file and a Google Sheet) on a single worksheet, **without** joining them at the row level.

### When to use Data Blending:
1.  **Cross-Database Limitations:** When you are using an older version of Tableau that doesn't support cross-database joins.
2.  **Different Granularities:** When your tables are at different levels of detail (e.g., `Sales` is tracked daily, but `Budget Targets` are tracked monthly). A standard join would duplicate your monthly budget target for every single day.

### How Data Blending Works:
* Data Blending acts like a dynamic **Left Join**. 
* **Primary Data Source (Blue Checkmark):** The first data source you drag onto your worksheet becomes the Primary. It dictates the main structure of the view.
* **Secondary Data Source (Orange Checkmark):** Tableau aggregates the data from the Secondary source *first*, and then maps those aggregated values to the matching dimensions in the Primary source.
* **Linking Fields:** Tableau uses a link icon (🔗) in the data pane to show which dimensions are being used to blend the two sources together.
