# Tableau: Dimensions vs. Measures & Calculated Fields

## Overview
Understanding how Tableau classifies data is the most critical step in mastering the software. Every field from your connected data source is automatically assigned a role as either a **Dimension** or a **Measure**, and as either **Discrete** (Blue) or **Continuous** (Green). This document covers these foundational concepts, along with how to manipulate this data using **Calculated Fields**.

---

## 1. Dimensions vs. Measures

When you connect to data, Tableau divides your fields into two main sections in the Data pane:

### Dimensions
* **Definition:** Dimensions contain qualitative values (such as names, dates, or geographical data). You can use dimensions to categorize, segment, and reveal the details in your data.
* **Function in View:** Dimensions affect the *level of detail* (granularity) in the view. 
* **Examples:** `Customer Name`, `Segment`, `Order Date`, `State`.
* **Visual Behavior:** When you drag a dimension to the Rows or Columns shelf, Tableau typically creates **Headers**.

### Measures
* **Definition:** Measures contain numeric, quantitative values that you can measure. 
* **Function in View:** Measures can be aggregated. By default, when you drag a measure into a view, Tableau applies an aggregation to it (e.g., `SUM(Sales)`, `AVG(Profit)`).
* **Examples:** `Sales`, `Profit`, `Quantity`, `Discount`.
* **Visual Behavior:** When you drag a measure to the Rows or Columns shelf, Tableau typically creates **Continuous Axes**.

---

## 2. Blue Pills vs. Green Pills (Discrete vs. Continuous)

In Tableau, fields are represented as "pills" when placed on shelves. The color of the pill is crucial: it does **not** indicate whether a field is a dimension or measure; it indicates whether it is **Discrete** or **Continuous**.

* **Discrete (Blue Pills):** * "Discrete" means individually separate and distinct.
  * Blue pills create **Labels / Headers**.
  * They slice the data into separate categories.
  * *Example:* A discrete `Order Date` will group data by distinct years (2021, 2022, 2023).

* **Continuous (Green Pills):**
  * "Continuous" means forming an unbroken whole, without interruption.
  * Green pills create **Axes**.
  * They plot data along a continuous scale.
  * *Example:* A continuous `Order Date` will plot data on a timeline from the earliest to the latest date.

*(Note: While Dimensions are usually discrete/blue and Measures are usually continuous/green, you can convert them. For example, you can have a Continuous Dimension for dates, or a Discrete Measure to group by specific numeric bins.)*

## 3. Calculated Fields in Tableau

Calculated fields allow you to create new data from data that already exists in your data source. They are written using a formula language native to Tableau (very similar to SQL and Excel formulas).

### When to use Calculated Fields:
* To segment data (e.g., creating custom groupings).
* To convert the data type of a field.
* To aggregate data uniquely.
* To filter out unwanted results.
* To calculate ratios (e.g., Profit Ratio).

---

## 4. Common Calculated Field Functions

Below are examples of standard calculated fields used in data analytics workflows. 

### A. Logical Calculations (IF / THEN / ELSE)
Used to categorize data conditionally.

```sql
// Example: Creating a Profitability Category
IF [Profit] > 0 THEN 'Profitable'
ELSEIF [Profit] = 0 THEN 'Break Even'
ELSE 'Unprofitable'
END
```

### B. String Manipulations
Used to clean, split, or format text data.

```sql
// Example: Extracting the first 3 letters of a Customer Name
LEFT([Customer Name], 3)

// Example: Combining (Concatenating) First and Last Name
[First Name] + " " + [Last Name]

// Example: Converting text to completely Uppercase
UPPER([City])
```

### C. Date Calculations
Used to measure time between dates or shift dates.

```sql
// Example: Calculating Shipping Delay in Days
DATEDIFF('day', [Order Date], [Ship Date])

// Example: Adding 30 days to an Order Date to set an SLA deadline
DATEADD('day', 30, [Order Date])

// Example: Extracting just the Year from a date
YEAR([Order Date])
```

### D. Mathematical Calculations
Basic arithmetic applied to your measures.

```sql
// Example: Calculating Total Cost from Sales and Profit
[Sales] - [Profit]
```

## 5. Row-Level vs. Aggregate Calculations

This is a critical concept in Tableau that often trips up beginners. You must understand the difference between calculating at the row level versus calculating at the aggregate level.

### Row-Level Calculation
The calculation evaluates **every single row** of the underlying data first, and then sums the result in the view.
* *Example:* `[Sales] / [Quantity]`
* *What happens:* Tableau goes to Row 1, divides Sales by Quantity. Goes to Row 2, does the same. Then it adds all those individual ratios together. (Usually, this yields mathematically incorrect totals for ratios).

### Aggregate Calculation
The calculation sums up all the data first, and then performs the math on the totals.
* *Example:* `SUM([Sales]) / SUM([Quantity])`
* *What happens:* Tableau adds up all the Sales for the selected dimension, adds up all the Quantities for that dimension, and *then* divides the total sales by the total quantity. (This is the correct way to calculate ratios like Profit Margin).

**Profit Ratio Example:**
```sql
// CORRECT: Aggregate Calculation
SUM([Profit]) / SUM([Sales])
```

---

## 6. Introduction to Table Calculations
Table calculations are a secondary type of calculation. While standard calculated fields are computed at the data source level, **Table Calculations** are computed locally in Tableau based *only* on what is currently in the view (the table).

* **Quick Table Calculations:** Tableau provides pre-built options accessible by right-clicking a measure pill in the view.
* **Common Types:**
  * Running Total
  * Percent of Total
  * Difference From (e.g., Month-over-Month growth)
  * Moving Average
  * Rank
