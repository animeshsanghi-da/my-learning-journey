# Business Intelligence: Tableau Dashboards & Stories

## Overview
While individual worksheets in Tableau allow you to explore data and build standard visualizations, **Dashboards** and **Stories** are where you bring those insights together to create interactive, cohesive data narratives. Mastering these features is essential for delivering actionable insights to stakeholders.

---

## 1. Introduction to Dashboards
A dashboard is a collection of several views (worksheets) and objects on a single canvas, allowing you to compare a variety of metrics and data simultaneously.

### Dashboard Objects and Layouts
* **Tiled vs. Floating:** * *Tiled:* Objects are arranged in a grid, automatically snapping to fill the available space. Best for responsive or standardized, clean layouts.
  * *Floating:* Objects can be layered on top of each other and freely positioned. Great for custom designs and putting legends inside charts, but requires manual adjustments for different screen resolutions.
* **Containers:** Use Horizontal and Vertical layout containers to group related worksheets and objects together. This allows them to resize dynamically when users interact with filters or when sheets are hidden.

---

## 2. Adding Interactivity with Dashboard Actions
Actions are the secret to building dynamic BI tools rather than static reports. They allow the dashboard to respond to user clicks, hovers, and selections.

### Types of Actions:
1. **Filter Actions:** Clicking on a mark in one sheet filters the data in other sheets. (e.g., Clicking "California" on a US Map filters the adjacent bar chart to show only California's sales).
2. **Highlight Actions:** Hovering or clicking a mark highlights related marks across all sheets while dimming the rest of the data.
3. **Go to Sheet Actions:** Navigates the user to another worksheet, dashboard, or story. Useful for creating a "Table of Contents" or navigation buttons.
4. **URL Actions:** Creates a hyperlink that points to a web page or file outside of Tableau, using data from your dashboard as dynamic URL parameters.

### Creating a Parameter for Dynamic Dashboards
Parameters allow users to input values that change the underlying calculations or filters across the entire dashboard.

```sql
// Example: Creating a Dynamic Metric Selector using a Parameter

// Step 1: Create a Parameter called [Select Metric] with String values: "Sales", "Profit", "Quantity"

// Step 2: Create a Calculated Field to swap the metric based on the user's Parameter selection
CASE [Select Metric]
    WHEN "Sales" THEN [Sales]
    WHEN "Profit" THEN [Profit]
    WHEN "Quantity" THEN [Quantity]
END

// Step 3: Drag this Calculated Field into your views. 
// When the user changes the parameter dropdown on the dashboard, all charts will update to show the newly selected metric.
```

## 3. Creating Tableau Stories
A Story is a sequence of visualizations that work together to convey information. You can create stories to tell a data narrative, provide context, demonstrate how decisions relate to outcomes, or make a compelling business case.

### Key Components:
* **Story Points:** Individual frames in a story. Each point can contain one worksheet or one entire dashboard.
* **Captions:** The navigation boxes at the top of the story. They should be used to describe the key takeaway, finding, or question for that specific story point.
* **Statefulness:** If you apply a filter or highlight a mark in a story point, Tableau saves that exact state. This allows you to use the *same* dashboard across multiple story points, but with different filters applied to show different angles of the data in a step-by-step presentation.

---

## 4. Best Practices for Dashboards & Stories
* **The 5-Second Rule:** A user should understand the main purpose of the dashboard within 5 seconds of looking at it.
* **Top-Left to Bottom-Right:** Users read screens from top-left to bottom-right. Place your most important, high-level KPIs in the top left, and granular, detailed tables in the bottom right.
* **Minimize Clutter:** Remove unnecessary gridlines, borders, and redundant legends. Let the data breathe by using whitespace effectively (padding).
* **Consistent Sizing:** Use "Fixed Size" for dashboards when you want exact control over the layout, ensuring your design won't break or stretch on different monitor sizes.
* **Clean Tooltips:** Always format your tooltips. A messy default tooltip ruins the professional feel of a dashboard.

```text
// Example of a Clean, Formatted Tooltip Layout
<Region> Region Performance
----------------------------
Total Sales:  <SUM(Sales)>
Total Profit: <SUM(Profit)>
Profit Ratio: <AGG(Profit Ratio)>
```
