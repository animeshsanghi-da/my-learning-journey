# File I/O: Reading, Writing, and Appending Files

## Overview
In Data Analytics and Software Development, your Python scripts rarely operate on hardcoded data alone. You need to read data from external sources (like text files, logs, or CSVs), process it, and output the results to new files. Python provides built-in functions to handle files seamlessly, making it an essential skill for building data pipelines and automating workflows.

---

## 1. The `open()` Function and File Modes
To work with a file in Python, you must first open it using the built-in `open()` function. This function returns a file object, which you then use to read or modify the file.

### Syntax
```python
file_object = open("filename.extension", "mode")
```

### Common File Modes
When opening a file, you specify a mode that dictates what operations are allowed.

| Mode | Name | Description |
| :--- | :--- | :--- |
| `'r'` | **Read** | Default mode. Opens a file for reading. Throws an error if the file doesn't exist. |
| `'w'` | **Write** | Opens a file for writing. **Creates a new file** if it doesn't exist, or **overwrites** the existing file entirely. |
| `'a'` | **Append** | Opens a file for appending. **Creates a new file** if it doesn't exist, or adds new data to the **end** of the existing file. |
| `'+'` | **Update** | Opens a file for updating (reading and writing). E.g., `'r+'`, `'w+'`. |

*(Note: Modes can be combined with `'t'` for text mode (default) or `'b'` for binary mode).*

---

## 2. Reading from a File (`'r'`)
When a file is opened in read mode, you can extract its contents using several methods depending on how you want to process the data.

### Methods for Reading:
* `read()`: Reads the entire file as a single string. You can also specify the number of characters to read (e.g., `read(5)`).
* `readline()`: Reads a single line from the file. Calling it again reads the next line.
* `readlines()`: Reads all lines and returns them as a **list of strings**, where each element is a line.

### Example: Reading a File
```python
# Open the file in read mode
f = open("sample_data.txt", "r")

# Read the entire content
content = f.read()
print("File Contents:\n", content)

# Always close the file to free up system resources
f.close()
```

### Iterating Line by Line (Memory Efficient)
For massive datasets, reading the entire file into memory at once is inefficient. Instead, iterate through the file object:
```python
f = open("large_log.txt", "r")
for line in f:
    print(line.strip()) # .strip() removes the trailing newline character
f.close()
```

---

## 3. Writing to a File (`'w'`)
Writing allows you to output data to a file. Remember: using `'w'` mode will **erase all existing content** in that file before writing the new data.

### Example: Writing Data
```python
# Open the file in write mode (creates it if it doesn't exist)
f = open("output.txt", "w")

# Write a string to the file
f.write("This is the first line of the new file.\n")
f.write("Data processing completed successfully.\n")

f.close()
```

### Writing Multiple Lines
You can use `writelines()` to write a list of strings to a file. Note that it does *not* automatically add newline characters; you must include them in your strings.
```python
lines_to_write = ["Data Point 1\n", "Data Point 2\n", "Data Point 3\n"]
f = open("output_list.txt", "w")
f.writelines(lines_to_write)
f.close()
```

---

## 4. Appending to a File (`'a'`)
If you want to add data to an existing file without deleting what is already there (e.g., adding new daily sales records to a master log), use append mode.

### Example: Appending Data
```python
# Open the file in append mode
f = open("output.txt", "a")

# Add new content to the end of the file
f.write("This line is appended to the existing text.\n")
f.write("System update logged at 10:00 AM.\n")

f.close()
```

---

## 5. The Importance of `.close()`
In the examples above, `f.close()` is called at the end of every block. 
1. **Resource Management:** Operating systems have a limit on how many files can be open simultaneously.
2. **Data Integrity:** When you write to a file, Python sometimes buffers the data in memory. Closing the file forces Python to flush the buffer and physically write the data to the hard drive. 

*Note: Forgetting to close files is a common source of bugs. In the next topic, we will explore the `with` statement, which automatically handles file closing for you.*
