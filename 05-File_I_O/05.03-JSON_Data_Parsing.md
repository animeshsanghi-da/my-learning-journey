# Parsing and Formatting JSON Data

## Overview
JSON (JavaScript Object Notation) is a lightweight data-interchange format. It is easy for humans to read and write, and easy for machines to parse and generate. In Data Science, Web Scraping, and API development, you will constantly encounter JSON when interacting with REST APIs, handling configuration files, or extracting data from NoSQL databases.

In Python, JSON data maps perfectly to Python dictionaries and lists. The built-in `json` module provides all the necessary methods to convert (serialize) Python objects to JSON and parse (deserialize) JSON strings back into Python objects.

---

## 1. The `json` Module
To work with JSON data in Python, you must first import the built-in module:
```python
import json
```

### Data Type Mapping (JSON to Python)
Understanding how data types translate between JSON and Python is critical:
* JSON `Object` `{}` -> Python `dict`
* JSON `Array` `[]` -> Python `list`
* JSON `String` `""` -> Python `str`
* JSON `Number` -> Python `int` or `float`
* JSON `true` / `false` -> Python `True` / `False`
* JSON `null` -> Python `None`

---

## 2. Parsing JSON (Deserialization)
Parsing means converting JSON formatted data (either a string or a file) into a Python dictionary or list so you can manipulate it with standard Python operations.

### Parsing from a String: `json.loads()`
If you have a JSON string (e.g., a text response from a web API), use `json.loads()` (which stands for "load string").

```python
import json

# A valid JSON string (note the double quotes around keys and strings, which are mandatory in JSON)
json_string = '{"name": "Animesh", "role": "Data Analyst", "skills": ["Python", "SQL", "Power BI"], "active": true}'

# Parse the JSON string into a Python dictionary
parsed_data = json.loads(json_string)

print(type(parsed_data))         # Output: <class 'dict'>
print(parsed_data["skills"][0])  # Output: Python
print(parsed_data["active"])     # Output: True (Notice lowercase 'true' became Python's capitalized 'True')
```

### Parsing from a File: `json.load()`
If your JSON data is stored in a physical `.json` file, use `json.load()` combined with Python's context manager (the `with` statement) to read it safely.

```python
import json

# Reading from a JSON file
with open("data.json", "r") as file:
    data = json.load(file)
    
print(data)
```

## 3. Formatting and Writing JSON (Serialization)
Serialization is the process of converting a Python dictionary or list into a JSON formatted string or file, usually to send data over a network or save it for later use.

### Converting to a String: `json.dumps()`
Use `json.dumps()` (which stands for "dump string") to convert a Python object into a JSON string.

```python
import json

# A standard Python dictionary
user_info = {
    "name": "Animesh",
    "age": 25,
    "certified": True,
    "projects": None
}

# Convert to JSON string
json_output = json.dumps(user_info)
print(json_output)
# Output: {"name": "Animesh", "age": 25, "certified": true, "projects": null}
```

### Pretty-Printing JSON
By default, `json.dumps()` outputs everything on a single line, which is hard to read. To make the JSON string human-readable, you can pass formatting arguments:
* `indent`: Specifies the number of spaces for indentation.
* `sort_keys`: Sorts the keys alphabetically.

```python
pretty_json = json.dumps(user_info, indent=4, sort_keys=True)
print(pretty_json)

/* Output:
{
    "age": 25,
    "certified": true,
    "name": "Animesh",
    "projects": null
}
*/
```

### Writing to a File: `json.dump()`
To save a Python dictionary directly to a `.json` file, use `json.dump()`.

```python
import json

data_to_save = {
    "dataset": "E-Commerce",
    "records": 100000,
    "status": "Cleaned"
}

# Writing the dictionary to a JSON file
with open("output.json", "w") as file:
    # We can also use 'indent' here for a nicely formatted, readable file
    json.dump(data_to_save, file, indent=4)
```

---

## 4. Common Errors & Troubleshooting
* **`json.decoder.JSONDecodeError`**: This is the most common error you will face. It happens if the JSON string/file is improperly formatted. Common causes include:
  * Using single quotes (`'`) instead of double quotes (`"`) for keys or strings (JSON strictly requires double quotes).
  * Having a trailing comma at the end of a JSON array or object (e.g., `[1, 2, 3,]`).
* **Type Conversion Nuances**: Remember that JSON requires keys to be strings. If you try to dump a Python dictionary with integer keys (e.g., `{1: "one"}`), they will automatically be converted to string keys (`{"1": "one"}`) in the resulting JSON.
