# NumPy: Creating Arrays & Memory Efficiency

## Overview
**NumPy** (Numerical Python) is the foundational package for scientific computing in Python. At its core is the `ndarray` (N-dimensional array) object. While standard Python lists are incredibly flexible, they are also memory-heavy and slow for large-scale numerical operations. NumPy solves this by using fixed-type, contiguous memory allocations (written in C), making it exponentially faster and much more memory-efficient.

---

## 1. Python Lists vs. NumPy Arrays (Memory & Speed)
Before creating arrays, it is crucial to understand *why* we use them. 

* **Python Lists:** A Python list is an array of pointers. Each element in the list is a separate Python object spread across memory, containing its own type information, reference count, and value. This scattered memory causes slow read/write speeds.
* **NumPy Arrays:** A NumPy `ndarray` stores elements of the *same data type* in a single, continuous block of memory. This allows modern CPUs to fetch data efficiently (vectorization) and eliminates the overhead of type-checking every individual element during operations.

By convention, NumPy is imported with the alias `np`.

## 3. Creating NumPy Arrays

### From Python Lists
The most basic way to create a NumPy array is by passing a Python list to `np.array()`.

```python
# 1D Array (Vector)
list_1d = [1, 2, 3, 4, 5]
arr_1d = np.array(list_1d)
print(arr_1d)

# 2D Array (Matrix) - Created from a list of lists
list_2d = [[1, 2, 3], [4, 5, 6]]
arr_2d = np.array(list_2d)
print(arr_2d)
```

### Built-in Array Creation Functions
Often, you need to initialize arrays with placeholder values without typing out entire lists.

```python
# np.zeros: Array filled with 0s (useful for initialization)
zeros_arr = np.zeros((3, 3))  # Creates a 3x3 matrix of zeros

# np.ones: Array filled with 1s
ones_arr = np.ones((2, 4))    # Creates a 2x4 matrix of ones

# np.full: Array filled with a specific constant value
full_arr = np.full((2, 2), 99) # Creates a 2x2 matrix filled with 99

# np.arange: Like Python's range(), but returns an array (start, stop, step)
range_arr = np.arange(0, 10, 2) # Output: [0, 2, 4, 6, 8]

# np.linspace: Creates an array of evenly spaced values over a specified interval
# Great for plotting graphs and mathematical modeling
linspace_arr = np.linspace(0, 1, 5) # Output: [0.  , 0.25, 0.5 , 0.75, 1.  ]

# np.empty: Allocates memory without initializing values (faster than zeros, but contains garbage values)
empty_arr = np.empty((2, 2))
```

### Creating Random Arrays
NumPy's `random` module is heavily used in machine learning for initializing weights and generating dummy datasets.

```python
# Random floats between 0 and 1
rand_floats = np.random.rand(3, 2)

# Random integers between a low (inclusive) and high (exclusive)
rand_ints = np.random.randint(1, 100, size=(3, 3))

# Random numbers from a standard normal distribution (mean=0, variance=1)
rand_normal = np.random.randn(1000) 
```

## 4. Inspecting Array Properties and Memory
To understand how an array is structured and how much memory it is consuming, we use object attributes (no parentheses needed).

```python
arr = np.array([[1, 2, 3], [4, 5, 6]], dtype=np.int32)

print(arr.ndim)     # Output: 2 (Number of dimensions)
print(arr.shape)    # Output: (2, 3) (Rows, Columns)
print(arr.size)     # Output: 6 (Total number of elements)
print(arr.dtype)    # Output: int32 (Data type of elements)

# Memory Efficiency Inspection
print(arr.itemsize) # Output: 4 (Memory size of one array element in bytes; int32 = 4 bytes)
print(arr.nbytes)   # Output: 24 (Total bytes consumed by the array's elements: 6 * 4)
```

## 5. Explicitly Declaring Data Types for Optimization
By default, NumPy might assign `int64` or `float64`. If you are working with millions of rows and you know your numbers won't exceed 255, you can force the array to use `int8` (1 byte per element instead of 8 bytes) to massively save memory.

```python
# Default usually float64
default_arr = np.array([1.1, 2.2, 3.3])
print(default_arr.nbytes) # 24 bytes (3 elements * 8 bytes)

# Optimized to float32
optimized_arr = np.array([1.1, 2.2, 3.3], dtype=np.float32)
print(optimized_arr.nbytes) # 12 bytes (3 elements * 4 bytes) - Memory cut in half!
```
NumPy supports a much greater variety of numerical types than Python does ([click here](https://numpy.org/doc/stable/user/basics.types.html)).
