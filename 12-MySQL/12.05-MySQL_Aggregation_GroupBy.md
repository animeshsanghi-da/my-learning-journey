# MySQL Data Aggregation: GROUP BY and HAVING

## Overview
Data aggregation is a fundamental skill for any Data Analyst. It involves grouping multiple rows of data together to perform calculations and return a single summary value. This module covers the essential tools for summarizing data in MySQL, including the `GROUP BY` clause, aggregate functions, and the `HAVING` clause.

---

## 1. Aggregate Functions
Aggregate functions perform a calculation on a set of values and return a single value. These are almost always used in conjunction with grouping.

### Common Aggregate Functions:
* **COUNT()**: Returns the number of rows that match a specific criterion.
* **SUM()**: Returns the total sum of a numeric column.
* **AVG()**: Returns the average value of a numeric column.
* **MIN()**: Returns the smallest value in a set.
* **MAX()**: Returns the largest value in a set.

### Usage Example:
```sql
-- Calculate basic statistics for the users table
SELECT 
    COUNT(*) AS total_users,
    SUM(salary) AS total_payroll,
    AVG(salary) AS average_salary,
    MIN(salary) AS lowest_salary,
    MAX(salary) AS highest_salary
FROM users;
```

---

## 2. The GROUP BY Clause
The `GROUP BY` statement groups rows that have the same values into summary rows. It is typically used with aggregate functions to find patterns across categories.

### How it Works:
1.  MySQL identifies all unique values in the specified column.
2.  It "buckets" all rows belonging to those unique values.
3.  It applies the aggregate function to each bucket.

### Example: Average Salary by Gender
```sql
SELECT gender, AVG(salary) AS average_salary
FROM users
GROUP BY gender;
```

### Example: Count of Referrals per User
```sql
SELECT referred_by_id, COUNT(*) AS total_referred
FROM users
WHERE referred_by_id IS NOT NULL
GROUP BY referred_by_id;
```

---

## 3. The HAVING Clause
The `HAVING` clause was added to SQL because the `WHERE` keyword cannot be used with aggregate functions. 

### Key Distinction:
* **WHERE**: Filters individual rows *before* they are grouped.
* **HAVING**: Filters groups *after* the aggregation has been performed.

### Example: Filter Groups by Aggregated Value
Suppose you want to see only those genders where the average salary is greater than 75,000:
```sql
SELECT gender, AVG(salary) AS avg_salary
FROM users
GROUP BY gender
HAVING AVG(salary) > 75000;
```

---

## 4. Advanced Grouping: WITH ROLLUP
The `WITH ROLLUP` modifier is used with `GROUP BY` to provide summary rows, such as subtotals and grand totals, within your result set.

### Example: Gender Count with Grand Total
```sql
SELECT gender, COUNT(*) AS total_users
FROM users
GROUP BY gender WITH ROLLUP;
```
*Note: The row where gender is `NULL` in the output represents the grand total of all users.*

---

## 5. Summary Table: WHERE vs. HAVING

| Clause | Purpose | Can Use Aggregates? |
| :--- | :--- | :--- |
| **WHERE** | Filters individual rows before grouping | No |
| **GROUP BY** | Groups rows based on column values | N/A |
| **HAVING** | Filters groups after aggregation | Yes |

---

## 6. Practical Workflow
When writing complex aggregation queries, follow this logical order:
1.  **SELECT**: Choose columns and aggregate functions.
2.  **FROM**: Define the source table.
3.  **WHERE**: Filter out unnecessary rows first.
4.  **GROUP BY**: Categorize the remaining data.
5.  **HAVING**: Filter the resulting categories based on the aggregate results.
6.  **ORDER BY**: Sort the final output for readability.
