# MySQL CRUD Operations: Create, Read, Update, and Delete

## Overview
CRUD is an acronym that stands for Create, Read, Update, and Delete. These are the four basic operations that a developer performs when interacting with a database. Understanding these is fundamental to managing data in any Relational Database Management System (RDBMS) like MySQL.

---

## 1. CREATE: Adding New Data
The `INSERT INTO` statement is used to add new rows of data to a table.

### Best Practice: Specifying Column Names
It is safer to specify the column names so that your query doesn't break if the table structure changes later.

```sql
-- Inserting a single record
INSERT INTO users (name, email, gender, date_of_birth) 
VALUES ('Bob', 'bob@example.com', 'Male', '1990-11-23');

-- Inserting multiple records at once (more efficient)
INSERT INTO users (name, email, gender, date_of_birth) VALUES
('Charlie', 'charlie@example.com', 'Other', '1988-02-17'),
('David', 'david@example.com', 'Male', '2000-08-09'),
('Eva', 'eva@example.com', 'Female', '1993-12-30');
```

**Note:** Columns like `id` (if set to `AUTO_INCREMENT`) or `created_at` (if set to `DEFAULT CURRENT_TIMESTAMP`) are handled automatically by MySQL.

---

## 2. READ: Querying Data
The `SELECT` statement is used to fetch data from a database.

### Basic Selection
```sql
-- Select everything from a table
SELECT * FROM users;

-- Select specific columns
SELECT name, email FROM users;
```

### Filtering with WHERE
The `WHERE` clause is used to extract only those records that fulfill a specified condition.

```sql
-- Filter by exact value
SELECT * FROM users WHERE gender = 'Male';

-- Filter using comparison operators
SELECT * FROM users WHERE id > 10;
SELECT * FROM users WHERE date_of_birth < '1995-01-01';

-- Filter for NULL values
SELECT * FROM users WHERE date_of_birth IS NULL;
```

### Sorting and Limiting
```sql
-- Sort results (ASC for ascending, DESC for descending)
SELECT * FROM users ORDER BY name DESC;

-- Limit the number of rows returned
SELECT * FROM users LIMIT 5;

-- Skip the first 5 rows and get the next 10 (Pagination)
SELECT * FROM users LIMIT 5, 10;
```

---

## 3. UPDATE: Modifying Existing Data
The `UPDATE` statement is used to modify the existing records in a table.

**CRITICAL WARNING:** Always use a `WHERE` clause when updating. If you omit it, **all** records in the table will be updated!

```sql
-- Update a single column for a specific record
UPDATE users 
SET name = 'Alicia' 
WHERE id = 1;

-- Update multiple columns
UPDATE users 
SET name = 'Robert', email = 'robert@example.com' 
WHERE id = 2;

-- Perform a calculation during update
UPDATE users 
SET salary = salary + 10000 
WHERE salary < 60000;
```

---

## 4. DELETE: Removing Data
The `DELETE` statement is used to remove existing records from a table.

**CRITICAL WARNING:** Like `UPDATE`, always use a `WHERE` clause to avoid deleting all your data!

```sql
-- Delete a specific row
DELETE FROM users WHERE id = 3;

-- Delete multiple rows based on a condition
DELETE FROM users WHERE gender = 'Other';

-- Delete all rows but keep the table structure
DELETE FROM users; 

-- Alternative: TRUNCATE is faster for deleting all rows
TRUNCATE TABLE users;
```

### Best Practices for Destructive Operations
* **Run a SELECT first:** Before running a `DELETE` or `UPDATE`, run a `SELECT` with the same `WHERE` clause to confirm exactly which rows will be affected.
* **Backup:** Always ensure you have a recent backup before performing bulk deletes or updates.
