# MySQL: Database Creation, Data Types, and Constraints

## 1. Introduction to DBMS
A Database Management System (DBMS) is software that interacts with users and applications to capture and analyze data. It allows for the creation, retrieval, updating, and management of structured data. MySQL is a widely-used relational DBMS that stores data in tables linked by relationships[.

---

## 2. Getting Started
### Creating and Using a Database
In MySQL, a database acts as a container for related tables, similar to an Excel workbook containing multiple sheets.

```sql
-- Create a new database
CREATE DATABASE startersql;

-- Select the database to use
USE startersql;
```

### Dropping a Database
To delete an entire database and all its contents:
```sql
DROP DATABASE startersql; -- Warning: This action is permanent 
```

## 3. Creating Tables
Tables store individual records (rows) consisting of specific attributes (columns).

```sql
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    gender ENUM('Male', 'Female', 'Other'),
    date_of_birth DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

## 4. Data Types Explained
MySQL uses specific data types to define what kind of information can be stored in a column:
* **INT**: Integer type for whole numbers.
* **VARCHAR(size)**: Variable-length strings up to a specified size.
* **DECIMAL(total, decimal)**: Exact numeric values, ideal for financial data (e.g., `DECIMAL(10,2)` allows 10 digits total with 2 after the decimal).
* **ENUM**: A string object with a value chosen from a permitted list.
* **DATE**: Stores date values in 'YYYY-MM-DD' format.
* **TIMESTAMP**: Stores date and time; often used to track when a row was created or modified.
* **BOOLEAN**: Stores TRUE or FALSE values.

## 5. Constraints
Constraints are rules applied to columns to ensure data accuracy, validity, and integrity.

| Constraint | Purpose | Example Syntax |
| :--- | :--- | :--- |
| **PRIMARY KEY** | Uniquely identifies each row; cannot be NULL. | `id INT PRIMARY KEY` |
| **NOT NULL** | Prevents a column from having a NULL value. | `name VARCHAR(100) NOT NULL` |
| **UNIQUE** | Ensures all values in a column are different. | `email VARCHAR(100) UNIQUE` |
| **AUTO_INCREMENT** | Generates a unique number automatically for new rows. | `id INT AUTO_INCREMENT` |
| **DEFAULT** | Sets a value if none is provided during insertion. | `is_active BOOLEAN DEFAULT TRUE` |
| **CHECK** | Ensures values satisfy a specific condition. | `CHECK (age >= 18)` |
| **FOREIGN KEY** | Links two tables together by referencing a Primary Key in another table. | `FOREIGN KEY (user_id) REFERENCES users(id)` |

## 6. Altering and Modifying Tables
You can use `ALTER TABLE` to change the structure of an existing table.

### Managing Columns
```sql
-- Add a new column
ALTER TABLE users ADD COLUMN city VARCHAR(100);

-- Drop an existing column
ALTER TABLE users DROP COLUMN city;

-- Modify a column's data type
ALTER TABLE users MODIFY COLUMN name VARCHAR(150);

-- Change a column name and type
ALTER TABLE users CHANGE COLUMN city location VARCHAR(150);
```

### Managing Constraints After Creation
```sql
-- Add a Primary Key later
ALTER TABLE users ADD PRIMARY KEY (id);

-- Add a Unique constraint
ALTER TABLE users ADD CONSTRAINT unique_email UNIQUE (email);

-- Add a Foreign Key

ADD CONSTRAINT fk_user 
FOREIGN KEY (user_id) REFERENCES users(id);
```

## 7. Primary Key vs. Unique
While both prevent duplicates, they serve different roles in database design:

| Feature | PRIMARY KEY | UNIQUE |
| :--- | :--- | :--- |
| **NULL Values** | Not allowed. | Allowed (one or more). |
| **Quantity** | Only one per table. | Multiple allowed per table. |
| **Purpose** | Main row identifier. | Enforce unique data like emails. |
