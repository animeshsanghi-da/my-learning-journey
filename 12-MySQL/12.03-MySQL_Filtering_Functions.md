# MySQL Filtering and Functions

## Overview
Filtering is the process of narrowing down a dataset to specific rows that meet certain criteria, while functions allow you to transform and summarize that data. These skills are fundamental for any Data Analyst using SQL to extract insights from relational databases.

---

## 1. Advanced Filtering with WHERE
The `WHERE` clause is used to filter records. Beyond simple equality, MySQL supports a variety of operators for complex logic.

### Comparison Operators
```sql
-- Equal to and Not Equal to
SELECT * FROM users WHERE gender = 'Male'; 
SELECT * FROM users WHERE gender != 'Female'; -- or <> 

-- Greater than and Less than
SELECT * FROM users WHERE id > 10; 
SELECT * FROM users WHERE date_of_birth < '1995-01-01'; 

-- Range and Set filtering
SELECT * FROM users WHERE date_of_birth BETWEEN '1990-01-01' AND '2000-12-31'; 
SELECT * FROM users WHERE gender IN ('Male', 'Other'); 
```

### Pattern Matching (Wildcards)
Wildcards are used with the `LIKE` operator to search for patterns in text columns.
* `%` : Matches any sequence of characters (zero or more).
* `_` : Matches exactly one single character.

```sql
SELECT * FROM users WHERE name LIKE 'A%';    -- Starts with A 
SELECT * FROM users WHERE name LIKE '%a';    -- Ends with a 
SELECT * FROM users WHERE name LIKE '%li%';  -- Contains 'li' 
SELECT * FROM users WHERE name LIKE '_a%';   -- Second letter is 'a' 
```

### Handling NULL Values
In SQL, `NULL` represents missing or unknown data. It cannot be compared using `=`, instead, use specific keywords.
```sql
SELECT * FROM users WHERE date_of_birth IS NULL; 
SELECT * FROM users WHERE date_of_birth IS NOT NULL; 
```

## 2. Built-in MySQL Functions
Functions transform your data directly within the query.

### Aggregate Functions
These return a single value calculated from a set of rows.
* **COUNT()**: Returns the number of rows.
* **SUM()**: Returns the total sum of a numeric column.
* **AVG()**: Returns the average value.
* **MIN() / MAX()**: Returns the lowest and highest values.

```sql
SELECT COUNT(*) AS total_users FROM users; 
SELECT AVG(salary) AS average_salary FROM users; 
SELECT MIN(salary) AS min_pay, MAX(salary) AS max_pay FROM users; 
```

### String Functions
Used to manipulate text data.
```sql
SELECT UPPER(name) FROM users;                 -- Convert to uppercase 
SELECT LENGTH(name) FROM users;                -- Get character count 
SELECT CONCAT(name, ' <', email, '>') FROM users; -- Combine columns 
```

### Date Functions
Crucial for time-series analysis and age calculations.
```sql
SELECT NOW();                                  -- Current date and time 
SELECT YEAR(date_of_birth) FROM users;         -- Extract year 
SELECT DATEDIFF(CURDATE(), date_of_birth);     -- Days since birth 
SELECT TIMESTAMPDIFF(YEAR, date_of_birth, CURDATE()) AS age FROM users; 
```

### Mathematical and Conditional Functions
```sql
-- Rounding
SELECT ROUND(salary) FROM users; 
SELECT FLOOR(salary), CEIL(salary) FROM users; 

-- Conditional Logic (Inline IF)
-- IF(condition, value_if_true, value_if_false)
SELECT name, IF(gender = 'Female', 'Yes', 'No') AS is_female FROM users; 
```

## 3. Organizing Results: ORDER BY and LIMIT
After filtering and transforming data, you often need to organize the output.

### ORDER BY
Sorts the result-set in ascending (`ASC`) or descending (`DESC`) order.
```sql
SELECT * FROM users ORDER BY name ASC; 
SELECT * FROM users ORDER BY salary DESC; 
```

### LIMIT and OFFSET
`LIMIT` restricts the number of rows returned, while `OFFSET` skips a specified number of rows. This is essential for pagination.
```sql
SELECT * FROM users LIMIT 5;              -- Top 5 rows 
SELECT * FROM users LIMIT 10 OFFSET 5;    -- Skip first 5, get next 10 
SELECT * FROM users LIMIT 5, 10;          -- Alternative syntax (skip 5, get 10) 
```
