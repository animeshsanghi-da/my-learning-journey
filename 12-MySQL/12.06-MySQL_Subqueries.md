# MySQL Advanced Querying: Subqueries

## Overview
A subquery is a query nested inside another SQL query (the outer query). Subqueries are powerful tools for breaking down complex problems into smaller, logical parts. They allow you to filter data based on dynamic or computed conditions that are not immediately available in a single table.

---

## 1. Types of Subqueries

### Scalar Subqueries
A scalar subquery returns exactly one value (one row and one column). These are typically used in `SELECT` lists or `WHERE` clauses for comparisons.

**Example: Find users earning more than the average salary**
```sql
SELECT id, name, salary
FROM users
WHERE salary > (
    SELECT AVG(salary) FROM users
);
```
*Explanation:* The inner query calculates the average salary once, and the outer query uses that single value to filter the users.

### Row Subqueries
A row subquery returns a single row with multiple columns. This allows you to compare multiple values at once.

### Multiple-Row Subqueries
These return a result set consisting of one column but multiple rows. They are frequently used with operators like `IN`, `ANY`, or `ALL`.

**Example: Find users referred by high-earning individuals**
```sql
SELECT id, name, referred_by_id
FROM users
WHERE referred_by_id IN (
    SELECT id FROM users WHERE salary > 75000
);
```
*Explanation:* The inner query returns a list of IDs of people earning > 75,000. The outer query finds anyone whose `referred_by_id` matches a value in that list.

---

## 2. Subqueries in Different Clauses

### In the WHERE Clause
This is the most common use case, used to filter the records of the main query based on the result of the inner query.

### In the SELECT Clause (Expression Subqueries)
You can use a subquery to return a specific calculated value for every row in the main result set.

**Example: Compare individual salary vs. company average**
```sql
SELECT name, salary,
(SELECT AVG(salary) FROM users) AS overall_avg
FROM users;
```

### In the FROM Clause (Derived Tables)
When a subquery is used in the `FROM` clause, the result acts as a temporary virtual table. 
*Note:* In MySQL, every derived table must have its own alias.

**Example:**
```sql
SELECT AVG(sub.total_salary) 
FROM (
    SELECT gender, SUM(salary) AS total_salary 
    FROM users 
    GROUP BY gender
) AS sub;
```

---

## 3. Correlated vs. Non-Correlated Subqueries

### Non-Correlated Subqueries
The inner query is independent of the outer query. It can be run on its own and provides a result that the outer query uses once. Most of the examples above are non-correlated.

### Correlated Subqueries
The inner query relies on data from the current row being processed by the outer query. This means the subquery executes once for every row processed by the outer query.

**Example: Find users who earn more than the average in their specific gender**
```sql
SELECT name, gender, salary
FROM users u1
WHERE salary > (
    SELECT AVG(salary) 
    FROM users u2 
    WHERE u2.gender = u1.gender
);
```

---

## 4. EXISTS and NOT EXISTS
The `EXISTS` operator is used to test for the existence of any record in a subquery. It returns `TRUE` if the subquery returns one or more records.

**Example: Find users who have at least one recorded address**
```sql
SELECT name FROM users
WHERE EXISTS (
    SELECT 1 FROM addresses WHERE addresses.user_id = users.id
);
```

---

## 5. Subquery vs. JOIN
While subqueries are often easier to read for complex logic, `JOINs` are usually more performant in MySQL because the execution engine can optimize them better. As a best practice, if a query can be written as a `JOIN`, it is generally preferred over a subquery for large datasets.
