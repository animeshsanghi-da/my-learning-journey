# MySQL Joins and Set Operations

## Overview
In relational databases, data is often spread across multiple tables to reduce redundancy. **JOINs** allow you to combine rows from these tables based on a related column (typically a Foreign Key). **SET Operations** (like UNION) allow you to combine the results of two different queries into a single result set.

---

## 1. SQL JOINs
JOINs are fundamental for retrieving complete information. For the examples below, assume we have a `users` table and an `addresses` table linked by `user_id`.



### INNER JOIN
Returns only the records that have matching values in both tables.
```sql
SELECT users.name, addresses.city
FROM users
INNER JOIN addresses ON users.id = addresses.user_id;
```
* **Behavior**: If a user does not have an address, or an address points to a non-existent user, those rows are excluded from the output.

### LEFT (OUTER) JOIN
Returns all records from the left table, and the matched records from the right table. If there is no match, the result is `NULL` on the right side.
```sql
SELECT users.name, addresses.city
FROM users
LEFT JOIN addresses ON users.id = addresses.user_id;
```
* **Use Case**: Finding all users, including those who haven't provided an address yet.

### RIGHT (OUTER) JOIN
Returns all records from the right table, and the matched records from the left table. If there is no match, the result is `NULL` on the left side.
```sql
SELECT users.name, addresses.city
FROM users
RIGHT JOIN addresses ON users.id = addresses.user_id;
```
* **Behavior**: Useful for identifying "orphaned" records in the child table that do not map to a parent.

### SELF JOIN
A regular join, but the table is joined with itself. This is useful for hierarchical data, such as a "Manager" or "Referrer" within an employee/user table.
```sql
-- Getting each user's name along with the name of the person who referred them
SELECT a.name AS user_name, b.name AS referred_by
FROM users a
INNER JOIN users b ON a.referred_by_id = b.id;
```
* **Pro Tip**: Use table aliases (like `a` and `b`) to differentiate the two instances of the same table.

## 2. SET Operations: UNION and UNION ALL
While JOINs combine columns horizontally, SET operations combine rows vertically from two or more `SELECT` statements.

### UNION
Combines the result set of two or more SELECT statements and **removes duplicates** by default.
```sql
SELECT name FROM users
UNION
SELECT name FROM admin_users;
```

### UNION ALL
Combines result sets but **keeps all duplicates**. It is faster than UNION because it doesn't perform the distinct check.
```sql
SELECT name FROM users
UNION ALL
SELECT name FROM admin_users;
```

### Rules for using UNION:
1. Each `SELECT` statement must have the same number of columns.
2. The columns must have compatible data types.
3. The columns must be in the same order.

## 3. Advanced Comparison Table
| Feature | INNER JOIN | LEFT JOIN | UNION |
| :--- | :--- | :--- | :--- |
| **Direction** | Horizontal (Adds columns) | Horizontal (Adds columns) | Vertical (Adds rows) |
| **Requirement** | Matching keys in both tables | Match not required for Left | Identical column counts |
| **Duplicates** | Based on Join logic | Based on Join logic | Removed (unless ALL used) |
