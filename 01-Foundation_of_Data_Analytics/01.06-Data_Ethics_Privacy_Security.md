# Data Ethics, Privacy, and Security

## Overview
As data analysts, we have a profound responsibility to handle data ethically and securely. This module, heavily emphasized in the **Google Data Analytics Professional Certificate**, covers the moral and legal obligations associated with data collection, storage, and analysis. It ensures that the insights we generate do not cause harm, discriminate, or violate user privacy.

---

## 1. Data Ethics & Principles
Data ethics refers to well-founded standards of right and wrong that dictate how data is collected, shared, and used. 

### Key Ethical Concepts:
* **Ownership:** Individuals own their data. Analysts and companies are merely custodians.
* **Transaction Transparency:** Subjects should understand how their data is being collected and what it will be used for.
* **Consent:** Explicit permission must be granted by the user before data collection occurs.
* **Currency:** Financial transactions derived from data should be fair and legal.
* **Openness (Open Data):** Free access, usage, and sharing of data (where appropriate and non-private) to promote transparency and public good.

---

## 2. Data Privacy vs. Data Security
While often used interchangeably, these two concepts handle different aspects of data protection.

* **Data Privacy:** Concerned with proper handling, processing, storage, and usage of personal information. It revolves around the *user's rights* (e.g., GDPR in Europe, CCPA in California). It answers the question: *Should we have access to this data?*
* **Data Security:** Concerned with protecting data from unauthorized access, breaches, or cyberattacks. It revolves around the *technical safeguards* (e.g., encryption, passwords, firewalls). It answers the question: *Is this data safe from hackers?*

---

## 3. Personally Identifiable Information (PII)
PII is any information that can be used by an organization on its own or with other information to identify, contact, or locate a single person.

* **Examples of PII:** Full name, Social Security Number, financial records, medical records (protected under HIPAA), phone numbers, and physical addresses.
* **Handling PII:** Analysts must practice **De-identification**. This includes:
  * **Anonymization:** Completely destroying the identifiable link so it can never be traced back.
  * **Pseudonymization:** Replacing identifiable data with artificial identifiers or pseudonyms. The link can be restored only with an additional secure key.

---

## 4. Mitigating Data Bias
Bias in data can lead to unfair or discriminatory outcomes. Recognizing and mitigating bias is a core ethical responsibility.

* **Sampling Bias:** When a sample is not representative of the population as a whole.
* **Observer Bias:** When an analyst's own beliefs or expectations influence their observation and data recording.
* **Confirmation Bias:** The tendency to search for, interpret, or favor information that confirms one's pre-existing beliefs.
* **Historical Bias:** When socio-cultural prejudices from the past are embedded into the data, causing algorithms to replicate those prejudices.

---

## 5. Practical Implementation: Securing and Masking Data
As an analyst utilizing Python and SQL, you will often need to mask or encrypt PII before sharing datasets or performing analysis.

### Example 1: Masking PII in SQL
Instead of exposing full credit card numbers or emails, we can use SQL string functions to mask them.

```sql
-- Masking an email address to show only the first 2 letters and the domain
SELECT 
    id,
    CONCAT(LEFT(email, 2), '***@', SUBSTRING_INDEX(email, '@', -1)) AS masked_email
FROM users;

-- Masking a phone number to show only the last 4 digits
SELECT 
    id,
    CONCAT('***-***-', RIGHT(phone_number, 4)) AS masked_phone
FROM users;
```

### Example 2: Hashing PII in Python
If you are preparing a dataset in Pandas and need to pseudonymize user IDs or emails, you can use Python's built-in `hashlib` to create secure hashes.

```python
import pandas as pd
import hashlib

# Sample Data
data = {'user_id': [1, 2, 3], 'email': ['alice@example.com', 'bob@example.com', 'charlie@example.com']}
df = pd.DataFrame(data)

# Function to hash text using SHA-256
def hash_data(text):
    # Encode text to bytes, hash it, and return the hexadecimal string
    return hashlib.sha256(text.encode('utf-8')).hexdigest()

# Apply hashing to the email column to secure PII
df['hashed_email'] = df['email'].apply(hash_data)

# Drop the original PII column before exporting or sharing the dataset
df_secure = df.drop(columns=['email'])

print(df_secure)
```
